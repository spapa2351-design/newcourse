## 🔴 Critical — 즉시 결정 필요

### GAP-01. 정원 마감 후 대기자 플로우 없음

현재 정원 초과 시 "마감" 배지만 표시하고 끝.  
**대기 신청 → 취소 발생 → 자동 알림 → 순번 확정** 플로우가 전혀 없다.

**문제점**
- 학습자는 "마감"을 보고 다른 차수를 신청해야 하는데 그 차수도 마감이면? → 이탈
- 취소가 생겨도 운영자가 수동으로 확인해야 잔여석을 알 수 있음
- 법정의무교육은 **전원 이수**가 목표 → 대기자 관리 없으면 미수료자 발생 필연

**결정 필요**
- [ ] 대기자 기능 도입 여부
- [ ] 대기 → 전환: 자동(선착순) vs 수동(관리자 승인)
- [ ] 대기 알림 채널 (앱 푸시 / 이메일 / SMS)

---

### GAP-02. 강제 할당 후 학습자 앱 노출 시점 미정

관리자가 명단을 등록해도 학습자 앱에 **"언제"** 보이는지 정의가 없다.  
퍼블리싱 전 미리 할당하는 케이스가 실무에서 가장 흔함.

| 시나리오 | 동작 |
|----------|------|
| A — 즉시 노출 | 할당 즉시 학습자 앱에 노출 (퍼블리싱 전에도 학습자가 볼 수 있음) |
| B — 퍼블리싱 시 노출 | 퍼블리싱 완료 후 일괄 노출 (준비 완료 후 공개 가능) |

**결정 필요**
- [ ] A / B 중 선택
- [ ] 강제 할당된 학습자가 취소를 원할 때 허용 여부 (자율 수강과 다른 규칙 필요)

---

### GAP-03. 알림 채널·타이밍 설정 없음

admin1.html에 "신청된 학습자에게 변경 알림이 발송됩니다" 문구만 있고 실제 알림 설정 화면이 없다.

**현재 미정인 알림 목록**
- 코스 신규 오픈 알림 — 어떤 대상에게? (전체 / 그룹 지정)
- 교육 D-3, D-1 리마인더 — 자동 발송 설정 없음
- 일정·장소 변경 통보 — 학습자 동의/거부 수신 여부
- 채널: 앱 푸시 / 이메일 / SMS — 어떤 걸 쓰는지 미정

**결정 필요**
- [ ] 알림 채널 확정
- [ ] 코스 빌더에 "알림 설정" 섹션 추가 여부

---

## 🟠 Important — 다음 스프린트 전 결정

### GAP-04. 지각 인정 기준 미정

수료 조건에 "지각도 출석으로 인정"이라고 되어 있는데, **지각의 시간 기준을 관리자가 설정할 수 없다.**

현재 QR 스캔 시간만 기록되고, 몇 분 이내를 "지각"으로 볼지 시스템이 모름.  
교통 지연으로 10분 늦은 경우 vs 1시간 늦은 경우가 동일하게 "지각"으로 처리됨 → 관리자가 매번 수동 판단.

**결정 필요**
- [ ] 지각 기준 시간을 코스/차수별로 설정 가능하게 할지
- [ ] 기준 초과 시 자동으로 "결석" 처리할지, 관리자 판단에 맡길지

---

### GAP-05. 공결(사전 결석) 처리 플로우 없음

출장·병가 등으로 사전 결석하는 케이스가 현실에서 가장 많이 발생하지만, 이 플로우가 없다.

**현재 없는 것들**
- 학습자가 사전에 결석 사유를 제출하는 경로
- 관리자가 결석을 "공결"로 분류해 수료 인정하는 기능
- 공결 시 다른 차수에서 보강하는 플로우

현재는 관리자가 admin1-1.html에서 수동으로 출석 상태를 "출석"으로 바꾸는 방법만 가능.  
→ 사유 기록 없이 조작 흔적만 남음

---

### GAP-06. 재수강 · 중복 신청 규칙 없음

두 가지 상충하는 케이스가 동시에 존재한다.

| 케이스 | 상황 | 필요한 것 |
|--------|------|-----------|
| A — 재수강 허용 | 법정의무교육은 매년 동일 코스 재이수 필요 | 재수강 차단하면 법정 의무 불이행 |
| B — 중복 차단 | 11/01 신청 + 11/02도 신청 → 정원 낭비 | 동일 코스 내 중복 신청 방지 |

**결정 필요**
- [ ] 코스별 "재수강 허용 여부" 토글 설정 방식 도입 여부
- [ ] 같은 코스 내 여러 차수 동시 신청 허용 여부

---

### GAP-07. 차수 일정 변경 시 기존 수강생 처리 미정

관리자가 이미 수강생이 있는 차수의 날짜·장소를 바꿨을 때 어떤 플로우로 처리하는지 미정.

**미정인 질문들**
- 기존 신청자에게 변경 통보 후 동의/거부를 받는가?
- 거부 시 자동으로 다른 차수로 이동하는가, 취소 처리하는가?
- 장소만 변경 vs 날짜 변경 — 동의 요구 수준이 다른가?

---

### GAP-08. 차수별 강사 설정 없음 · 강사 권한 미정

코스 전체에 강사 1명만 지정 가능. 실제 운영에서는 차수(반)별로 다른 강사가 배정되는 경우가 다수.

> **케이스 (프로젝트 문서 기반)**  
> 전국 150개 매장 점장 교육 → 서울/부산/광주 3개 권역 동시 진행 → 권역마다 다른 강사

**미정인 것들**
- 강사 계정이 출석 QR을 띄울 수 있는지 (시스템 접근 권한)
- 강사가 수강생 명단을 조회할 수 있는지

## 🟡 Minor — 이후 결정 가능

### GAP-09. 수료 후 연계 트리거 미정

수료 처리가 완료됐을 때 자동으로 일어나야 하는 것들이 연결되지 않았음.

- 수료증 자동 발급 (PDF 생성 → 앱 내 보관)
- 인사 시스템 연동 (C사 케이스: 학점 반영, B사 케이스: 고과 반영)
- 학습자 수료 알림 발송

---

### GAP-10. 신청 완료 학습자 경험 미설계

학습자가 신청 완료 후 받아야 할 경험이 설계되지 않음.

- 확인 알림 (앱 푸시 / 이메일)
- 신청한 일정을 캘린더(Google Calendar / Outlook)에 추가하는 기능
- 취소 시 UX: 확인 모달 → 취소 사유 입력 → 완료 메시지

---

### GAP-11. "차수" 용어 충돌 미확정

프로젝트 문서에 이미 경고 기록됨.

- 내부 정의: 차수 = 반/기수 (A반, 1기)
- 일부 고객사(C사) 정의: 차수 = 회차(session)

학습자 앱에 노출될 최종 명칭 미확정.

---

### GAP-12. 미수료자 관리 뷰 부재

프로젝트 문서에서 이미 갭으로 기록됨.

- 전체 대상자 중 미수료자 추출 뷰
- 차수 걸친 코스 전체 수료율 집계 (예: 8개 차수 합산)
- 법정의무교육 증빙용 미수료자 리스트 export

## PM 권고 · 1순위

**GAP-01 대기자 플로우를 먼저 결정하세요.**

정원 마감 코스가 생기는 순간 운영 혼란이 즉시 발생합니다. 대기자 기능 없이 오픈하면 학습자는 이탈하고, 운영자는 수동으로 관리해야 합니다. 법정의무교육처럼 **전원 이수가 목표인 코스**에서는 특히 치명적입니다.

| 옵션 | 동작 |
|------|------|
| 선착순 자동 전환 | 취소 발생 → 대기 1순위 자동 배정 → 알림 발송 |
| 관리자 수동 전환 | 취소 발생 → 관리자에게 알림 → 수동으로 대기자 배정 |
| 대기 없음 (Phase 1) | 마감 시 다른 차수 선택 유도 + 오픈 시 알림 받기만 지원 |
