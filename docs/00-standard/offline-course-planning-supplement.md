# 1. 운영 정책 (Operations Policy)

> 이 섹션은 운영팀의 매뉴얼이자, 개발팀의 구현 기준이며, 영업팀의 고객 안내 근거입니다.
> 모든 정책은 이 문서에 기록된 후에만 공식입니다. 슬랙·구두 합의는 여기 반영 전까지 비공식입니다.

## 1.1 출석 상태 정의 및 전환 규칙

### 상태 정의

| 상태 | 시스템 키 | 의미 | 부여 방식 | 아이콘/색상 |
|------|-----------|------|-----------|-------------|
| 미확인 | `pending` | 회차 생성 시 모든 수강생의 초기 상태. 아직 출석 여부가 확인되지 않은 임시 상태 | 시스템 자동 | 회색 |
| 출석 | `attended` | 수업 시작 시각 기준, 지각 허용 시간 이내에 QR 스캔 완료 | QR 스캔 / 관리자 수동 | 녹색 |
| 지각 | `late` | 수업 시작 시각 + 지각 허용 시간 초과 후 QR 스캔 | QR 스캔 / 관리자 수동 | 주황색 |
| 조퇴 | `early_leave` | 출석 후 수업 종료 전 퇴실. 반드시 관리자가 직접 변경 | 관리자 수동 전용 | 노란색 |
| 결석 | `absent` | 해당 회차에 출석하지 않음. 운영자가 미확인 → 결석으로 직접 변경 | 관리자 수동 | 빨간색 |

### 상태 전환 매트릭스

| 현재 상태 → | 미확인 | 출석 | 지각 | 조퇴 | 결석 |
|------------|:------:|:----:|:----:|:----:|:----:|
| **미확인** | — | ✅ QR/수동 | ✅ QR/수동 | ❌ | ✅ 마감 일괄/수동 |
| **출석** | ❌ | — | ✅ 수동 | ✅ 수동 | ✅ 수동 |
| **지각** | ❌ | ✅ 수동 | — | ✅ 수동 | ✅ 수동 |
| **조퇴** | ❌ | ❌ | ❌ | — | ✅ 수동 |
| **결석** | ❌ | ✅ 수동 | ✅ 수동 | ❌ | — |

> **원칙**: 미확인 상태로 되돌리는 것은 불가. 조퇴에서 출석/지각으로 되돌리는 것도 불가.
> **이유**: 출석 이력의 신뢰성 보장. 조퇴는 물리적으로 퇴실한 상태이므로 소급 변경 불가.

## 1.2 지각 처리 정책

### 기준 설정

| 항목 | 규칙 |
|------|------|
| 설정 단위 | 차수별 (동일 코스라도 차수마다 다를 수 있음) |
| 설정 필드 | `lateMinutes` (분 단위 정수) |
| 기본값 | 10분 |
| 0 입력 시 | 지각 기준 없음 = QR 스캔 시각과 무관하게 모두 출석 처리 |
| 변경 가능 시점 | 차수 시작 전까지. 차수 진행 중에는 변경 불가 |

### 판정 로직

```
QR 스캔 시각 ≤ 수업 시작 시각 + lateMinutes → 출석
QR 스캔 시각 > 수업 시작 시각 + lateMinutes → 지각
```

### 지각 누적 규칙

| 규칙 | Phase 1 | Phase 2 예정 |
|------|---------|-------------|
| 지각 N회 = 결석 1회 환산 | ❌ 미적용 | 고객사 요구 시 추가 |
| 지각 시 수료 영향 | 지각은 출석으로 간주 (수료율 계산에 포함) | 별도 환산 비율 설정 가능 |

> **운영 참고**: Phase 1에서 지각은 수료율 계산 시 "출석"과 동일하게 취급됩니다.
> 지각 3회 = 결석 1회 같은 환산 규칙은 Phase 2에서 고객사별 설정으로 제공 예정입니다.

## 1.3 회차 종료 정책 (2026-05-06 마감 개념 제거 — 시간 기반 자동)

### 자동 종료 흐름

| 시점 | 행위자 | 일어나는 일 |
|------|--------|------------|
| 수업 종료 시각 | 시스템 | 회차 자동 종료 (별도 클릭 없음). 출석률 계산 시작. 미확인은 "출석 확인 중" 표시 |
| 종료 후 | 운영자 | 미확인 인원을 직접 정리 (출석/지각/결석 수동 변경, "미확인 일괄 처리" 도구 사용) |

### 회차 종료 규칙 (2026-05-06 마감 개념 제거 — 시간 기반 자동)

| 규칙 | 내용 |
|------|------|
| 회차 종료 시점 | **수업 종료 시각 자동** (별도 마감 버튼 없음) |
| 회차 종료 후 상태 변경 | 관리자 수동 변경 가능. 변경 로그 자동 기록 |
| "마감 자체 취소" | ❌ 개념 자체 없음 — 마감 사건이 없어 풀 일도 없음 |
| 미확인 처리 | 운영자가 직접 정리. "미확인 일괄 처리" 도구 사용 (전원 출석/전원 결석/개별) |
| 학습자 알림 | Phase 1: **즉시성만** (C1 출석 처리됨, C2 출석률 표시). Phase 2 이월: C3 미확인 처리 중 / C3 결석 확정 / C4 HR 시스템 연동 |
| 운영자 도구 | "미확인 일괄 처리" — 전원 출석 / 전원 결석 / 개별 처리 (마감 버튼 대체) |

> **운영 주의**: 회차 종료 = 수업 종료 시각. 별도 클릭 없이 자동 처리. 미확인 인원은 운영자가 직접 정리.
>
> **Phase 1 범위 밖 (Phase 2 이월)**: 자동 결석 처리 / 운영자 단계별 알림 (24h/72h/6일/7일) / 시스템 잠금 (30일 회계 마감) / "자동 결석 처리 OFF" 토글 — 모두 Phase 2에서 검토.

## 1.4 수료 처리 정책

### 수료 조건

| 항목 | Phase 1 기준 |
|------|-------------|
| 기본 수료 기준 | 전체 회차 중 출석률 75% 이상 |
| 출석으로 인정되는 상태 | `attended`, `late`, `early_leave` |
| 출석으로 인정되지 않는 상태 | `absent`, `pending` |
| 수료 판정 시점 | 모든 회차 종료 완료 후 (운영자가 미확인 인원 정리 후), HRD 담당자가 수동으로 "수료 처리" 버튼 클릭 |

### 수료 처리 프로세스

| 단계 | 행위자 | 행동 | 시스템 반응 |
|------|--------|------|------------|
| 1 | 시스템 | 전 회차 종료 여부 확인 (운영자가 미확인 인원 정리 후) | 진행중 회차 있으면 수료 처리 버튼 비활성 |
| 2 | HRD 담당자 | "수료 처리" 버튼 클릭 | 수료 대상자 목록 표시 (75% 이상 충족자) |
| 3 | HRD 담당자 | 확인 클릭 | 대상자 일괄 수료 확정, 미충족자는 미수료 상태 유지 |

### 수료 확정 후 규칙

| 규칙 | 내용 |
|------|------|
| 수료 확정 후 출석 변경 | 가능. 단, 수료 상태는 자동 재계산되지 않음 |
| 수료 취소 | Phase 1 미지원. 수료 확정은 최종 |
| 수료 기준 변경 | 코스 개설 시 설정. 차수 진행 중 변경 불가 |

> **운영 주의**: 수료 처리는 1회성입니다. 처리 전 출석 데이터가 정확한지 반드시 확인하세요.

## 1.5 수강 신청 정책

### 신청 유형

| 유형 | 설명 | 적용 사례 |
|------|------|-----------|
| 자율 수강 | 학습자가 직접 신청. 신청 기간 내 선착순 | 자기개발 교육, 선택 교육 |
| 강제 할당 | HRD 담당자가 대상자를 직접 배정 | 법정의무교육, 직급별 필수교육 |
| 혼합 | 자율 + 강제 병행. 일부는 할당, 나머지는 자율 신청 | 필수 대상자 + 추가 희망자 |

### 정원 관리

| 규칙 | 내용 |
|------|------|
| 정원 초과 | 신청 불가. "마감" 배지 표시 + 다른 차수 안내 |
| 대기자 | Phase 1 미지원. 다른 차수 유도 |
| 정원 변경 | 차수 시작 전까지 변경 가능 |

### 재수강 규칙

| 규칙 | 내용 |
|------|------|
| 재수강 허용 여부 | 코스별 토글 설정 (기본값: 허용 안 함) |
| 동일 차수 중복 신청 | 항상 불가 |
| 다른 차수 중복 신청 | 재수강 허용 시에만 가능 |
| 법정의무교육 | 재수강 허용 필수 (매년 재이수) |

### 신청 취소·변경

| 규칙 | 내용 |
|------|------|
| 취소 가능 기한 | 코스 개설 시 설정 (예: 교육 시작 D-3일 전까지) |
| 기한 이후 취소 | 학습자 직접 취소 불가. HRD 담당자만 처리 가능 |
| 차수 변경 | Phase 1 미지원. 기존 취소 + 새 차수 신청으로 처리 |

## FLOW-01: 코스 개설 (HRD 담당자)

### 목적
새로운 오프라인 교육 과정을 시스템에 등록합니다.

### 전제 조건
- HRD 담당자 계정으로 로그인
- 교육 일정, 장소, 강사가 사전에 확정됨

### 흐름

| 순서 | 행동 | 결과 | 소요 시간 |
|:----:|------|------|:---------:|
| 1 | 관리자 화면 > "새 코스 등록" 클릭 | 코스 등록 화면 진입 | — |
| 2 | 기본정보 입력 (코스명, 카테고리, 강사명, 교육장소, 표지 이미지) | 기본정보 섹션 완료 표시 | 3분 |
| 3 | 수강 신청 설정 (자율/강제, 신청 기간, 취소 정책, 재수강 허용) | 신청설정 섹션 완료 표시 | 2분 |
| 4 | 수료 조건 설정 (출석률 기준) | 수료조건 섹션 완료 표시 | 1분 |
| 5 | 차수 생성 (날짜, 정원, 담당 강사, 지각 기준) | 차수 카드 생성 | 3분 |
| 6 | 회차 생성 (각 수업일 + 시간) — 수동 또는 일괄 생성 | 회차 목록 생성 | 2~5분 |
| 7 | 퍼블리싱 체크리스트 확인 | 8항목 모두 통과 시 퍼블리싱 가능 | 1분 |
| 8 | "퍼블리싱" 클릭 | 학습자 앱에 교육 과정 노출 | — |

### 예외 상황

| 상황 | 시스템 대응 |
|------|------------|
| 필수 항목 누락 | 퍼블리싱 버튼 비활성 + 미완료 섹션 안내 |
| 차수 없이 퍼블리싱 시도 | "최소 1개 차수를 생성하세요" 안내 |
| 회차 없는 차수 존재 | "회차가 없는 차수가 있습니다" 경고 |

## FLOW-02: 수강 신청 (학습자 — 자율 수강)

### 목적
학습자가 원하는 교육 과정에 직접 신청합니다.

### 전제 조건
- 학습자 앱 로그인
- 해당 코스가 퍼블리싱 완료 상태
- 신청 기간 내

### 흐름

| 순서 | 행동 | 결과 |
|:----:|------|------|
| 1 | 교육 목록에서 원하는 코스 선택 | 코스 상세 화면 진입 |
| 2 | 차수 목록 확인 (날짜, 시간, 잔여석) | 신청 가능 차수 확인 |
| 3 | 원하는 차수의 "신청" 버튼 클릭 | 신청 확인 다이얼로그 |
| 4 | 확인 | 신청 완료. 잔여석 -1 반영 |

### 예외 상황

| 상황 | 시스템 대응 |
|------|------------|
| 정원 마감 | "마감" 배지 표시. 다른 차수 안내 |
| 신청 기간 외 | 신청 버튼 비활성. "신청 기간: MM/DD ~ MM/DD" 표시 |
| 이미 신청한 차수 | "이미 신청한 교육입니다" 안내 |
| 재수강 비허용 코스에 재신청 | "이미 수료한 교육입니다" 안내 |

## FLOW-03: 수강생 강제 할당 (HRD 담당자)

### 목적
법정의무교육 등 필수 교육 대상자를 관리자가 직접 배정합니다.

### 흐름

| 순서 | 행동 | 결과 |
|:----:|------|------|
| 1 | 코스 등록 화면 > 수강 신청 유형 "강제 할당" 선택 | 그룹 배정 UI 활성화 |
| 2 | 대상 그룹(부서/직급) 선택 | 해당 그룹 인원이 차수에 자동 배정 |
| 3 | 정원이 자동 합산됨 | 배정 인원 = 차수 정원 |
| 4 | 퍼블리싱 | 대상자 앱에 즉시 노출 |

### 예외 상황

| 상황 | 시스템 대응 |
|------|------------|
| 그룹 인원이 정원 초과 | 경고 표시 + 정원 자동 조정 확인 |

## FLOW-04: 현장 출석 (강사 / HRD 담당자)

### 목적
교육 당일, 수강생의 출석을 QR 코드로 확인합니다.

### 흐름

| 순서 | 행동 | 결과 |
|:----:|------|------|
| 1 | 관리자 > 해당 코스 > 출석 관리 진입 | 출석 관리 화면 |
| 2 | 해당 회차 선택 > "라이브 모드" 클릭 | QR 코드 전체 화면 표시 (30초마다 자동 갱신) |
| 3 | 수강생이 모바일로 QR 스캔 | 실시간 출석 반영 (사이드바에 즉시 표시) |
| 4 | 수업 종료 시각 도래 | 회차 자동 종료 (별도 클릭 없음). 출석률 계산 시작 |
| 5 | 수업 종료 후 미확인 인원 정리 | 운영자가 직접 처리 ("미확인 일괄 처리" 도구 사용) |

### 예외 상황

| 상황 | 시스템 대응 |
|------|------------|
| QR 인식 실패 | 관리자가 수동으로 출석 처리 |
| 지각 허용 시간 초과 후 스캔 | 자동으로 "지각" 처리 |
| 다른 사람의 QR을 스캔 | Phase 1에서는 기술적 방지 불가. 운영 규칙으로 통제 |

## FLOW-05: 수료 처리 (HRD 담당자)

### 목적
차수의 모든 회차가 완료된 후, 수료 기준 충족자를 확정합니다.

### 전제 조건
- 해당 차수의 **모든 회차가 종료** 완료 (운영자가 미확인 인원 정리 후)

### 흐름

| 순서 | 행동 | 결과 |
|:----:|------|------|
| 1 | 출석 관리 화면 > 출석 탭 | 전 회차 출석 현황 확인 |
| 2 | 모든 회차 종료 여부 확인 (운영자가 미확인 인원 정리 후) | 진행중 회차 있으면 수료 처리 버튼 비활성 |
| 3 | "수료 처리" 클릭 | 수료 대상자 목록 표시 (75% 이상) |
| 4 | 확인 | 수료 확정. 미충족자는 미수료 상태 |

### 주의사항
- 수료 처리는 **1회성**입니다. 처리 전 모든 출석 데이터를 확인하세요.
- 수료 확정 후 출석을 수정해도 수료 상태는 자동 변경되지 않습니다.

## 4.1 고객의 현재 문제

### 지금 고객사는 이렇게 일합니다

```
1. 교육 기획 → 엑셀에 코스 정보 수동 입력
2. 수강 신청 → 이메일/전화로 접수, 엑셀에 명단 입력
3. 출석 체크 → 종이 서명부 또는 수동 명부 체크
4. 수료 처리 → 엑셀에서 출석률 수동 계산
5. 보고 → 엑셀 데이터 복사해서 보고서 작성
```

### 이 방식의 문제

| 문제 | 구체적 사례 | 영향 |
|------|-----------|------|
| **수동 오류** | 출석 체크 누락, 엑셀 수식 오류 | 수료율 부정확 → 감사 리스크 |
| **시간 낭비** | 매 교육마다 엑셀 복사·수정 | HRD 담당자 시간의 40% 이상 소비 |
| **데이터 분산** | 코스별 엑셀 파일이 각각 존재 | 전체 현황 파악 불가 |
| **추적 불가** | 미수료자 관리, 재교육 대상 추출 어려움 | 법정의무교육 미이수 리스크 |
| **보고 지연** | 수동 집계 → 보고서 작성 | 경영진 의사결정 지연 |

## 4.3 고객 대상별 핵심 메시지

### HRD 담당자에게

> "엑셀 없이, 코스 개설부터 수료 처리까지 한 화면에서 끝납니다.
> QR 출석으로 현장 오류가 사라지고, 수료율은 시스템이 자동 계산합니다."

### IT 담당자에게

## 5.1 서비스 구조 용어

| 용어 | 영문 | 의미 | 예시 | 비고 |
|------|------|------|------|------|
| **코스** | Course | 하나의 교육 프로그램. "무엇을 배우는가" | "2026년 기업윤리교육", "신입사원 리더십 과정" | 가장 큰 단위 |
| **커리큘럼** | Curriculum | 코스 안에서 "어떻게 배우는가"를 정의하는 학습 구성 | 오프라인 3회 + 온라인 사전학습 1회 | 코스 1개에 커리큘럼 1개 |
| **슬롯** | Slot | 커리큘럼 안의 개별 학습 단계. 오프라인 출석, 온라인 영상 시청 등 | "오프라인 출석 슬롯", "온라인 영상 슬롯" | 슬롯 타입에 따라 다른 동작 |
| **차수** | Session | 동일한 코스를 반복 운영하는 단위. "몇 번째 개설인가" | "2026년 상반기 1차수", "2026년 하반기 2차수" | 차수마다 날짜·정원·강사가 다를 수 있음 |
| **회차** | Class | 차수 안의 개별 수업일. "몇 번째 수업인가" | "1차수 1회차 (5/1)", "1차수 2회차 (5/8)" | 회차마다 출석을 체크 |

### 계층 관계 (한눈에 보기)

```
코스 "기업윤리교육"
 └─ 커리큘럼 (오프라인 출석 4회)
      └─ 슬롯: 오프라인 출석
           ├─ 1차수 (2026 상반기, 강사: 김민수, 정원 30명)
           │   ├─ 1회차 (5/1 09:00)
           │   ├─ 2회차 (5/8 09:00)
           │   ├─ 3회차 (5/15 09:00)
           │   └─ 4회차 (5/22 09:00)
           └─ 2차수 (2026 하반기, 강사: 이영희, 정원 25명)
               ├─ 1회차 (9/1 14:00)
               └─ ...
```

## 5.3 상태 용어

| **진행중** | 차수의 수업이 시작된 상태 | 코스 목록 |
| **완료** | 모든 차수의 수료 처리가 완료된 상태 | 코스 목록 |
