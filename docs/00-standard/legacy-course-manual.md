# 레거시 터치클래스 — 코스 매뉴얼 정리

> **출처**: `Touchclass_관리자_매뉴얼_v.15.pdf` (62~88p) + `Touchclass_학습자_매뉴얼_v.3.pdf` (10~13p)

> **추출 범위**: 코스 관련 부분만. 콘텐츠·문제은행·라이브 등 다른 영역은 의도적으로 제외.

## 1. 코스 목록 (관리자)

### 1.1 코스 목록 화면

- **썸네일 뷰** — 운영 상태, 코스명, 코스 표지, 수강 기간, 분류, 수강 인원, 등록일, 생성자, 연결 콘텐츠 표시
  - **운영 상태**: 운영중 / 오픈예정 / 종료
  - **콘텐츠 버튼**: 연결 콘텐츠의 에디터 화면으로 이동
  - **링크 복사**: 학습자에게 전달할 코스 링크
  - **수정**: 코스 분류·코스명·수강 기간 등 수정
  - **삭제**
- **리스트 뷰** — 기능은 썸네일 뷰와 동일
- **다운로드** — 코스 정보 파일(엑셀) 다운로드
- **코스 정보 일괄 수정** — 여러 코스 한 번에:
  - 코스 분류 (추가·변경·삭제)
  - 수강 등록 방식 + 그룹
  - 수강 기간 / 관리자 종료 / 자리비움 모니터링 시간
  - 필수 코스 설정 (진도율·학습시간·수료증 발급)
  - 페이지 순서대로 수강 / 수강 전 코스 정보 보기

### 1.2 코스 분류

- 분류 관리(⚙) → 등록·수정·삭제·순서 변경
- 화면·기능은 콘텐츠 분류와 동일

### 1.3 코스 순서 관리

- 드래그앤드롭으로 순서 변경
- 순서 변경 시 사용자 코스 목록의 **추천순 정렬**에 반영
- "종료 코스 숨기기 ON" → 종료 코스 제외 상태에서 순서 변경

## 2. 코스 등록 (관리자)

### 2.1 콘텐츠 선택 (필수, 1단계)

- 코스에 사용할 콘텐츠를 먼저 선택
- ❗ **코스 개설 후에는 콘텐츠 변경 불가**

### 2.2 코스 기본 정보

- 코스 분류 선택
- 코스명 입력
- 코스 소개 입력
- **코스 상세 상단**: 학습자 코스 상세 화면 상단에 표시 — **최대 5장 이미지** 설정 가능

### 2.3 수강 등록 방식

두 모드를 **각각 ON/OFF** 가능 (병행 운영 가능):

#### ① 회원이 신청
- 회원이 수강 신청 후 수강
- **수강 신청 기간** 설정
- **수강 신청 권한** (그룹 지정)

#### ② 관리자가 등록
- 관리자가 수강 등록 후 수강
- ❗ 관리자 수강 등록 그룹에 회원 추가 시 **자동으로 코스에 수강 등록**

### 2.4 수강 기간 지정

- ① **시작일~종료일 지정**: 지정 기간 동안 학습 가능
- ② **수강 시작일로부터 N일간**: 수강생마다 등록 시점부터 N일간
- ③ **제한 없음**: 등록 시점부터 종료일 제한 없이

❗ 수강 기간 수정 시, 기존 수강생 수강 기간 변경 여부 선택 가능

### 2.5 수강 시간 제한

- 설정한 **요일·시간** 동안에만 수강 가능
- 제한 시간엔 학습자가 코스 진입 불가

### 2.6 코스 상세 정보 (5개 옵션)

#### ① 수강 시간 확인을 위한 자리 비움 모니터링
- 학습 화면에서 특정 시간 동안 학습 활동이 없으면 학습시간 측정 정지
- 학습 재개 시 수강 로그 새로 추가

#### ② 커리큘럼별 공개일 설정 (드립 학습)
- 커리큘럼별 오픈일 설정
- **공개일 전까지 숨김** vs **공개일 전에 미리 보여줌**
- **공개일 기준** 라디오:
  - 수강 등록일로부터 적용 (개인별)
  - 코스 수강 시작일로부터 적용 (전체 일괄)

#### ③ 페이지 순서대로 수강하기
- 커리큘럼 및 페이지를 **순차적으로 학습**해야 함

#### ④ 필수 코스로 지정
- '필수' 표시 노출
- 설정한 조건 충족 시 '학습 완료' 표시
- 충족 조건들:
  - **수료증 발급** (조건 충족 시 자동 발급)
  - **진도율** ❗ 콘텐츠 페이지를 보기만 해도 진도율 계산
  - **학습시간** ❗ **페이지 내 동영상 재생 시간과 무관** (재생 안 해도 페이지 머무르기로 충족) — *정합성 깨짐 주의*
  - **시험** — 점수 기준
    - 재응시 기능 (점수 미만 시 재시험)
    - 콘텐츠에 '결과' 요소 포함 시에만 필수 조건 설정 가능
  - **설문** — 제출 기준
  - **동영상 학습 조건** — 동영상별 재생률 기준 (예: 20분 영상의 80%인 16분 재생)
    - **skip 제한** — 동영상 스킵 차단

❗ 미제출/점수 미달 시 다음 페이지 이동 제한 = '페이지 순서대로 수강하기' ON일 때만 가능
❗ 영상 재생률 미충족 시 다음 페이지 이동 제한 = '페이지 순서대로 수강하기' ON일 때만 가능

#### ⑤ 수강 전 코스 정보 보기
- 학습자가 코스 목록에서 코스 선택 시 '코스 상세 정보' → 페이지로 이동

## 3. 수강생 등록

### 3.1 등록 방법 (4가지)

- **회원**: 전체 회원 또는 회원 선택
- **그룹**: 그룹 선택
- **대량 등록**: 회원 ID 입력 csv 파일 업로드
- **아이디 입력**: 회원 아이디 직접 입력

❗ 4가지 방법 모두 **1개 이상의 코스에 한 번에 등록 가능** (코스 다중 선택)

## 4. 수강 관리 (관리자)

### 4.1 요약

- ① **코스 정보**: 수강 등록 인원 / 학습 인원 / 필수 학습 완료 인원 / 평균 진도율 / 평균 학습 시간
- ② **코스 퀴즈/시험/설문 정보**: 문항 수 / 평균 정답률 / 만점(시험)
- ③ **그룹별 학습량**: 평균 진도율 / 평균 학습 시간 / 총 학습 시간

### 4.2 운영 제어

- **종료하기/시작하기** — 운영 중인 코스 중단·재개
  - ❗ 수강 기간은 변경되지 않음
  - 운영 종료된 코스는 학습자에게 비표시
- **코스 QR 코드** — 학습자가 코스로 이동하는 QR·링크

### 4.3 수강생 정보 (일반 코스)

- ① **진도율** — 콘텐츠 페이지 기준 학습 진도율
- ② **수강 기간** — 개별 수정 가능, '전체 수강 기간 변경'으로 일괄 수정
- ③ **학습시간** — 페이지 학습 시간
- ④ **상세** — 개별 수강 정보

### 4.4 수강생 정보 (필수 코스)

학습 완료 조건 표시:
- ① **최종 완료** — 모든 조건 충족 여부
- ② **수료 일자** — 최종 완료 일자
- ③ **진도율** + 조건 충족 여부
- ④ **학습시간** + 조건 충족 여부
- ⑤ **동영상** — 재생 개수 + 조건 충족 여부
- ⑥ **시험** — 점수 + 조건 충족 여부
- ⑦ **설문** — 제출 + 조건 충족 여부

다운로드:
- **수료증 다운로드** (개별/일괄)
- **수료 현황 다운로드**

❗ 시험·설문 충족 여부는 콘텐츠의 **'결과' 요소** 기준
- '결과' 버튼 클릭 안 하면 충족 여부 표시 X
- '결과' 요소가 2개 이상이면 위치별로 풀이·응답 현황 표시

### 4.5 동영상

- 수강생의 **동영상 재생 시간·재생률** 확인
- 회원별 동영상 재생 시간 다운로드

### 4.6 퀴즈

- ① 응시 / 정답 / 오답 수
- ② 답안 초기화 (수강생별 또는 전체)
- 응시 여부 검색 가능

### 4.7 시험·설문

- ① 시험 점수 / 응시 / 정답 / 오답 (점수는 시험만)
- ② 설문 응답 / 미응답
- ③ 답안(응답) 초기화 (수강생별 또는 전체)
- ④ 시험 결과 공개 / 설문 종료

❗ 응시했지만 미제출 = '미응시' 필터에서 확인

### 4.8 PDF·이미지

- **PDF**: 조회 횟수 / 다운로드 횟수
- **이미지**: 클릭하여 외부·내부 링크 이동 횟수

### 4.9 의견

- 코스 등록 의견·댓글 확인·관리
- 기간별 / 그룹별 / 이름·아이디별 조회
- 관리자 댓글 작성 가능
- 수정·삭제·블라인드

## 5. PLUS 코스 (터치콘텐츠 서비스 사용 시만)

❗ PLUS 코스는 **삭제 불가**

### 5.1 차이점 (vs 일반 코스)

- 콘텐츠는 **PLUS 콘텐츠** 선택
- 코스 분류·코스명·소개 자동 입력 (수정 가능)
- 수강 기간 옵션 2개만: 시작일~종료일 / 수강 시작일로부터 N일간 (제한 없음 X)
- 수강 등록 방식·코스 상세 정보는 자체 코스와 동일
- 수강생 삭제 불가
- 수강 기간 개별 수정 불가

## 10. NEW 코스 설계 시 짚어야 할 정합성 이슈

레거시 메뉴얼에서 명시적으로 인정된 **정합성 결함**:

| 영역 | 레거시 동작 | 문제 |
|---|---|---|
| 학습시간 | 페이지 머무르기로 충족 (영상 재생 무관) | **영상 안 보고 페이지 열어두기로 100% 충족 가능** — 부정 학습 |
| 진도율 | 콘텐츠 페이지를 **보기만 해도** 카운트 | 실제 학습 안 해도 진도율 올라감 |
| 단위 | **페이지** = 영상·텍스트·시험 묶음 | 단위가 너무 커서 정밀 측정 불가 |

NEW 모델에서는:
- 페이지 단위 → **에셋(항목) 단위**
- 영상 = 실제 재생 시간 기준 시청률
- 시험 = 점수 기준
- 학습시간 = 실제 학습 활동 합산 (페이지 머무르기 X)

## 11. 용어 매핑 (레거시 → NEW)

| 레거시 | NEW | 비고 |
|---|---|---|
| 콘텐츠 | (없음 — 인라인 정의) | NEW는 코스에서 챕터-항목 직접 정의 |
| 페이지 | (해체됨) | 영상·시험·아티클 등 에셋 단위로 분해 |
| 커리큘럼 | 커리큘럼 (= 챕터) | 동일 |
| 일반 코스 / 필수 코스 | 일반 코스 / 필수 코스 | 동일 라벨 유지 (운영자 학습 곡선 X) |
| PLUS 코스 | (별도 영역, 향후 검토) | 터치콘텐츠 서비스 의존 |
| 페이지 순서대로 수강 | 전체 순서대로 수강 (일괄 매크로) + 항목별 prerequisite | NEW는 매크로 + 항목별 정밀 제어 |
| 수강 신청 / 관리자 등록 두 모드 동시 ON | 라디오 (택 1) | 운영 단순화 — 4인 합의 |
| 콘텐츠 결과 요소 | (없음 — 시험 자체에 점수·통과 기준) | 더 단순 |
