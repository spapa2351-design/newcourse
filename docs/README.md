## 1. Executive Summary

### 1.1 Phase 1 완결 현황

| 항목 | 내용 |
|------|------|
| **범위** | 오프라인 코스 목록 · 등록 · 수강관리 + 출석관리 + 온라인 코스 등록 |
| **방식** | Retrospective PDCA (기존 프로토타입 → 문서화 → Gap 분석 → 수정) |
| **결과** | 5개 파일 모두 Match Rate **100%** 달성 |
| **갭 처리** | 총 21개 갭 식별 → 17건 즉시 해결, 4건 Minor 이월 |
| **추가 기획 갭** | 12개 기획 구멍 발견 (gap-report.md) → 3 Critical · 5 Important · 4 Minor |

### 1.2 파일별 Match Rate

| 파일 | 기능 | 초기 | 최종 |
|------|------|:----:|:----:|
| `index.html` | 오프라인 코스 목록 | 93% | **100%** |
| `offline-builder.html` | 오프라인 코스 등록 빌더 | 82% | **100%** |
| `offline-course-view.html` | 코스 수강관리 | 81% | **100%** |
| `offline-attendance.html` | 출석·수료 관리 | 83% | **100%** |
| `online-builder.html` | 온라인 코스 등록 | 84% | **100%** |

### 1.3 핵심 가치

오프라인 교육 운영의 엑셀 의존을 제거하고, **코스 개설 → QR 출석 → 수료 처리** 전 과정을 단일 웹 시스템으로 통합. 빌드 없이 브라우저에서 즉시 실행 가능한 단일 HTML 파일 구조로 빠른 도입·검증이 가능.

## 2. 사용자 정의

### 2.1 주요 사용자

| 역할 | 주요 작업 | 접근 화면 |
|------|-----------|-----------|
| **HRD 담당자** | 코스 개설, 차수/회차 생성, 수강생 배정, 출석 확인, 수료 처리 | 관리자 전체 |
| **강사** | QR 출석 화면 운영, 출석 현황 확인 | offline-course-view.html (출석·수료 탭) |
| **학습자** | 수강 신청, QR 스캔으로 출석 | 학습자 앱 (별도) |
| **IT 담당자** | 시스템 배포, 관리자 계정 생성 | 서버/배포 환경 |

### 2.2 사용자별 Pain Point (Phase 1 이전)

| HRD 담당자 | 학습자 |
|------------|--------|
| 코스마다 엑셀 복사·수정 (업무 시간 40%+) | 이메일/전화로 신청 → 확인까지 수 시간 |
| 종이 서명부 출석 체크 → 수동 집계 | 자신의 출석 상태 실시간 확인 불가 |

| 전체 코스 현황 한눈에 파악 불가 | — |

## 3. 서비스 구조

### 3.1 파일 구성 (admin/)

| 파일 | 역할 | 상태 |
|------|------|------|
| `index.html` | 오프라인 코스 목록 대시보드 | Phase 1 완료 |
| `offline-builder.html` | 코스 개설/편집 빌더 (섹션 1~4 + 우측 커리큘럼 사이드바) | Phase 1 완료 |
| `offline-course-view.html` | 수강생 관리 · 출석 관리 · 수료 처리 | Phase 1 진행 중 (정식 화면 확정 2026-04-28) |
| `online-builder.html` | 온라인 코스 등록 빌더 | Phase 1 완료 |
| `offline-builder-v2.html` | 오프라인 등록 빌더 대안 (보조 참고용, 링크 제거됨) | 참고용 |

### 3.2 기술 스택

```
React 18 (Babel standalone, ESM CDN)  — 빌드 없이 브라우저 직접 실행
Tailwind CSS (CDN)
Lucide React 0.292.0
단일 HTML 파일 구조
```

### 3.3 계층 구조 (핵심 데이터 모델)

```
코스 (무엇을 배우나 — 제목·강사·분류·신청 설정)
 └── 커리큘럼 (어떻게 배우나 — 슬롯 순서/구성)
       └── 슬롯 (개별 학습 단계)
             ├── 오프라인 출석 슬롯  ← Phase 1 유일 슬롯 타입
             │     └── 차수 (운영 회기: 날짜·정원·강사)
             │           └── 회차 (개별 수업일)
             ├── 온라인 영상 슬롯   ← Phase 2 추가 예정
             ├── 온라인 시험 슬롯   ← Phase 2 추가 예정
             └── 온라인 설문 슬롯   ← Phase 2 추가 예정
```

**계층 예시**

```
코스 "기업윤리교육"
 └─ 커리큘럼 (오프라인 출석 4회)
      └─ 슬롯: 오프라인 출석
           ├─ 1차수 (2026 상반기, 강사: 김민수, 정원 30명)
           │   ├─ 1회차 (5/1 09:00)
           │   ├─ 2회차 (5/8 09:00)
           │   ├─ 3회차 (5/15 09:00)
           │   └─ 4회차 (5/22 09:00)
           └─ 2차수 (2026 하반기, 강사: 이영희, 정원 25명)
               ├─ 1회차 (9/1 14:00)
               └─ ...
```

### 3.4 네비게이션 링크 현황

- `index.html` → 코스 등록: `offline-builder.html`
- `index.html` → 코스명 클릭 / 편집 아이콘 / 차수 편집: `offline-builder.html?courseId=...`
- `offline-builder.html` → 목록: `index.html` · 수강관리: `offline-course-view.html`
- `online-builder.html` → 목록: `index.html`

### 3.5 글로벌 사이드바

`index.html`, `online-builder.html`, `offline-builder.html`, `offline-course-view.html` 4개 파일 모두 동일한 좌측 사이드바 적용.

- **구성**: 🔥 로고 + 17개 메뉴 항목 (회원, 코스, 오프라인 코스, 콘텐츠 … 팝업)
- **active 표시**: `offline-builder.html` / `offline-course-view.html` → 오프라인 코스 항목 하이라이트 · `online-builder.html` → 코스 서브메뉴 active

## 4. 운영 정책

> 이 섹션의 모든 정책은 공식 기준입니다. 슬랙·구두 합의는 이 문서에 반영 전까지 비공식입니다.

### 4.1 출석 상태 정의

| 상태 | 시스템 키 | 의미 | 부여 방식 | 색상 |
|------|-----------|------|-----------|------|
| 미확인 | `pending` | 회차 생성 시 전원 초기 상태. 아직 판단 전 임시 상태 | 시스템 자동 | 회색 |
| 출석 | `attended` | 수업 시작 + 지각 허용 시간 이내 QR 스캔 | QR 스캔 / 수동 | 녹색 |
| 지각 | `late` | 지각 허용 시간 초과 후 QR 스캔 | QR 스캔 / 수동 | 주황색 |
| 조퇴 | `early_leave` | 출석 후 수업 종료 전 퇴실. 반드시 관리자가 변경 | 수동 전용 | 노란색 |
| 결석 | `absent` | 해당 회차 미출석. 운영자가 수동으로 처리 (미확인 → 결석으로 직접 변경) | 수동 | 빨간색 |

### 4.2 상태 전환 매트릭스

| 현재 → | 미확인 | 출석 | 지각 | 조퇴 | 결석 |
|--------|:------:|:----:|:----:|:----:|:----:|
| **미확인** | — | ✅ QR/수동 | ✅ QR/수동 | ❌ | ✅ 마감 일괄/수동 |
| **출석** | ❌ | — | ✅ 수동 | ✅ 수동 | ✅ 수동 |
| **지각** | ❌ | ✅ 수동 | — | ✅ 수동 | ✅ 수동 |
| **조퇴** | ❌ | ❌ | ❌ | — | ✅ 수동 |
| **결석** | ❌ | ✅ 수동 | ✅ 수동 | ❌ | — |

> **원칙**: 미확인 상태로 되돌리는 것은 불가. 조퇴에서 출석/지각으로 되돌리는 것도 불가.

### 4.3 지각 처리 정책

| 항목 | 규칙 |
|------|------|
| 설정 단위 | 차수별 (`lateMinutes`, 분 단위 정수) |
| 기본값 | 10분 |
| 0 입력 시 | 지각 기준 없음 = 모두 출석 처리 |
| 변경 가능 시점 | 차수 시작 전까지 |
| 판정 로직 | QR 스캔 시각 ≤ 시작 + lateMinutes → 출석 / 초과 → 지각 |
| 수료 계산 | 지각은 출석으로 간주 (Phase 1) |

### 4.4 회차 종료 정책 (2026-05-06 마감 개념 제거 — 시간 기반 자동)

| 규칙 | 내용 |
|------|------|
| 회차 종료 시점 | **수업 종료 시각 자동** (별도 마감 버튼 없음) |
| 미확인 처리 | 운영자가 직접 정리 ("미확인 일괄 처리" 도구 사용). 자동 결석 처리·운영자 단계별 알림은 Phase 2 이월 |
| 회차 종료 후 상태 변경 | 가능. 관리자 무제한 정정. 변경 로그 자동 기록 |
| "마감 자체 취소" | 개념 자체 없음 — 마감 사건이 없어 풀 일도 없음 |
| 운영자 도구 | "미확인 일괄 처리" — 전원 출석 / 전원 결석 / 개별 처리 |
| 학습자 알림 (Phase 1) | 즉시성만 — 출석 처리됨 (QR 직후) / 출석률 표시. 결석 통보·HR 연동은 Phase 2 |

> **운영 주의**: 회차 종료는 시간이 결정합니다. 별도 클릭 없이 자동 처리. 미확인 인원은 운영자가 직접 정리.

### 4.5 수료 처리 정책

| 항목 | Phase 1 기준 |
|------|-------------|
| 기본 수료 기준 | 전체 회차 출석률 **75% 이상** |
| 출석으로 인정 | `attended`, `late`, `early_leave` |
| 출석으로 미인정 | `absent`, `pending` |
| 수료 판정 시점 | 모든 회차 종료 완료 후 (운영자가 미확인 인원 정리 후) "수료 처리" 버튼 클릭 |
| 수료 확정 후 출석 변경 | 가능. 단, 수료 상태 자동 재계산 안 됨 |
| 수료 취소 | Phase 1 미지원 |

> **운영 주의**: 수료 처리는 1회성입니다. 처리 전 출석 데이터가 정확한지 반드시 확인하세요.

### 4.6 수강 신청 정책

| 유형 | 설명 |
|------|------|
| 자율 수강 | 학습자 직접 신청. 신청 기간 내 선착순 |
| 강제 할당 | HRD 담당자가 그룹/부서 단위로 직접 배정 |

**정원 관리**

| 규칙 | 내용 |
|------|------|
| 정원 초과 | 신청 불가. "마감" 배지 + 다른 차수 안내 |
| 대기자 | Phase 1 미지원. 다른 차수 유도 |
| 동일 차수 중복 신청 | 항상 불가 |
| 재수강 | 코스별 토글 설정 (기본값: 허용 안 함). 법정의무교육은 허용 필수 |
| 취소 기한 이후 | 학습자 직접 취소 불가. HRD 담당자만 처리 가능 |

## 5. 사용자 흐름

### FLOW-01: 코스 개설 (HRD 담당자)

| 순서 | 행동 | 소요 시간 |
|:----:|------|:---------:|
| 1 | 관리자 > "새 코스 등록" 클릭 | — |
| 2 | 기본정보 입력 (코스명, 카테고리, 강사, 교육장소, 표지 이미지) | 3분 |
| 3 | 수강 신청 설정 (자율/강제, 신청 기간, 취소 정책, 재수강 허용) | 2분 |
| 4 | 수료 조건 설정 (출석률 기준) | 1분 |
| 5 | 차수 생성 (날짜, 정원, 강사, 지각 기준) | 3분 |
| 6 | 회차 생성 (각 수업일 + 시간) — 수동 또는 일괄 생성 | 2~5분 |

## 6. 기능 명세

### 6.1 코스 목록 (index.html)

- **헤더**: 코스 목록 + 상태 탭(전체/준비중/모집중/진행중/완료) + 카운트 뱃지
- **좌측 사이드바**: 분류 트리 (WORK > 직무역량/리더십/영업, 공통/온보딩, IT, DX, 기타)
- **메인 테이블 컬럼**: 체크 | 확장▶ | 상태 | 필수 | 표지 | 코스명 | 구성 | 분류 | 기간 | 수강인원 | 등록일 | 관리
- **확장 행(Expandable Row)**: 차수별 잔여석·일정 인라인 표시
- **검색**: 코스명/강사명 실시간 검색
- **삭제**: 단건 삭제 · 체크박스 다중 선택 삭제

### 6.2 오프라인 코스 등록 빌더 (offline-builder.html)

**레이아웃**: 좌측 메인(800px) + 우측 사이드바(500px) + 상단 헤더

**섹션 1: 코스 기본정보**
- 코스명, 코스 상세 상단(코스 표지 기본 / 이미지 최대 5장), 카테고리(5개 그룹), 강사, 강사소개, 코스소개, 교육 장소 설정(장소명/주소/교통안내)
- 섹션 완성도 표시: `SecStatus` (done/partial/empty)

**섹션 2: 수강 신청 설정**
- 수강 신청 기간 (오픈/마감 datetime)
- 수강 등록 방식: 자율 수강(신청권한 그룹, 단일/다중 선택) / 관리자 강제 할당(그룹 칩 배정)
- 취소 정책: 언제든 취소 / 기한부 취소 / 취소 불가
- 일정 변경 제한 토글

**섹션 3: 수강 신청 옵션 (차수) 생성**
- 차수 카드: 이름, 기간, 정원, 개별 장소, 지각 기준
- 강제 할당 모드: 그룹 칩 UI (정원 자동 계산)
- 차수 복사·삭제

**섹션 4: 수료 조건**
- Phase 1: 오프라인 출석 슬롯 고정 (필수, 편집 불가)
- 출석률 기준 설정 (기본값 75%)

**우측 사이드바 (커리큘럼 관리)**
- 미니 캘린더 (날짜 클릭 → 해당 회차로 이동)
- 차수별 회차 카드: 회차명, 날짜, 시작/종료 시간, 장소
- 회차 추가 · 일괄 생성 (요일+시간 입력 → 자동 생성)
- 차수/회차 DnD 정렬

**퍼블리싱 플로우**
1. 8항목 체크리스트 자동 검증
2. 공개 시점 선택 (즉시 / 예약)
3. 퍼블리싱 확정

### 6.3 수강생·출석 관리 (course-learners.html)

**좌측 사이드바**: 코스 표지, 코스명, 차수 드롭다운, 상태/정원/잔여석/회차, QR 코드(소형)

**탭 구조**: 요약 | 수강 | 출석

**요약 탭**
- KPI 카드 4개: 총인원, 출석인원(미출석 수), 평균출석률, 다음교육일
- 회차별 출석률 바 차트
- 차수별 비교 테이블

**수강 탭**
- 수강생 테이블: 상태 | 사진 | 이름 | 아이디 | 그룹 | 출석률 | 수강기간 | 관리
- 수강생 등록·삭제

**출석 탭**
- 회차 선택 칩 버튼 (출석률 표시)
- 출석 테이블: 상태(5종 뱃지) | 사진 | 이름 | 그룹 | 출석시간 | 관리
  - 미확인 상태: [출석] 원클릭 + [...] 드롭다운(지각/결석)
  - 확정 상태: 상태 변경 드롭다운
- 미확인 일괄 처리 도구 (전원 출석 / 전원 결석 / 개별) / 수료 처리 버튼 (회차 종료는 시간 기반 자동, 마감 버튼 없음)

**라이브 출석 모드 (전체화면)**
- 좌측 70%: QR 코드(대형) + 30초 카운트다운
- 우측 30%: 다크 대시보드 — 실시간 출석자 수(48px), 상태별 카운트, 최근 출석자 리스트

## 9. 비즈니스 맥락

### 9.1 고객의 현재 문제

```
1. 교육 기획 → 엑셀에 코스 정보 수동 입력
2. 수강 신청 → 이메일/전화로 접수, 엑셀에 명단 입력
3. 출석 체크 → 종이 서명부 또는 수동 명부 체크
4. 수료 처리 → 엑셀에서 출석률 수동 계산
5. 보고     → 엑셀 데이터 복사해서 보고서 작성
```

| 문제 | 구체적 사례 | 영향 |
|------|-----------|------|
| **수동 오류** | 출석 누락, 엑셀 수식 오류 | 수료율 부정확 → 감사 리스크 |
| **시간 낭비** | 매 교육마다 엑셀 복사·수정 | HRD 담당자 시간 40%+ 소비 |
| **데이터 분산** | 코스별 엑셀 파일 각각 존재 | 전체 현황 파악 불가 |
| **추적 불가** | 미수료자 관리, 재교육 대상 추출 어려움 | 법정의무교육 미이수 리스크 |

### 9.2 우리 서비스가 해결하는 것

| 고객 문제 | 솔루션 | 효과 |
|-----------|--------|------|
| 엑셀로 코스 관리 | 웹 화면에서 코스 개설~퍼블리싱 | 관리 시간 70% 감소 |
| 종이 서명부 | QR 코드 실시간 출석 | 출석 오류 제로 |
| 수동 수료 계산 | 시스템 자동 수료율 + 1클릭 수료 처리 | 정확한 수료율 보장 |
| 데이터 분산 | 단일 시스템에서 전체 코스 관리 | 전체 현황 즉시 파악 |

### 9.3 경쟁 대비 차별점

| 항목 | 기존 LMS | 우리 서비스 |
|------|---------|------------|
| 도입 기간 | 3~6개월 | 즉시 (빌드 불필요) |
| 오프라인 특화 | 온라인 중심, 오프라인은 부가 | 오프라인 운영 중심 설계 |
| QR 출석 | 별도 장비/앱 필요 | 모바일 브라우저만으로 가능 |
| 차수/회차 관리 | 단순 일정 관리 | 차수별 강사·지각기준·정원 개별 설정 |

### 9.4 고객 대상별 핵심 메시지

**HRD 담당자**: "엑셀 없이, 코스 개설부터 수료 처리까지 한 화면에서 끝납니다. QR 출석으로 현장 오류가 사라지고, 수료율은 시스템이 자동 계산합니다."

**IT 담당자**: "빌드 없이 브라우저에서 바로 실행됩니다. 단일 HTML 파일 구조라 기존 시스템에 임베딩이 쉽습니다."

**경영진**: "교육 투자 대비 효과를 실시간으로 확인할 수 있습니다. 법정의무교육 이수율 관리가 자동화되어 감사 리스크가 줄어듭니다."

```bash
git -C "J:/claude/newcourse" remote set-url origin "https://<PAT>@github.com/spapa2351-design/newcourse.git"
git -C "J:/claude/newcourse" push origin master
```

- 토큰 발급: GitHub → Settings → Developer settings → Fine-grained tokens

### 12.3 Phase 2 착수 시 최우선 결정 사항

Phase 2를 시작하기 전 반드시 다음 3개를 먼저 결정하세요.

1. **알림 채널 (GAP-03)**: 어떤 채널로 어떤 알림을 보낼 것인가? 결정 없이는 코스 빌더에 알림 설정 섹션을 추가할 수 없습니다.

2. **대기자 플로우 (GAP-01)**: 정원 마감 시 대기 신청을 받을 것인가? 법정의무교육처럼 전원 이수가 목표인 코스에서 치명적입니다.

3. **강제 할당 노출 시점 (GAP-02)**: 퍼블리싱 전에 할당된 학습자 앱에 코스가 보여야 하는가? 운영 정책 결정 없이 구현 불가.
