# 2026-05-19 디디쌤 온라인 강좌 등록 UI 차용 검토 (AI 제외)

비교 대상:
- 디디쌤 가이드 Chapter 3 (기간 클래스 만들기, AI SAM 항목 제외)
- newcourse 현재 "커리큘럼 관리" 빌더 화면 (스크린샷 첨부, 2026-05-19 기준)

## 비교 결론

**newcourse 빌더가 디디쌤보다 더 간단함.**

- 디디쌤 = 5스텝 멀티페이지 (클래스 정보 → 커리큘럼 → 수료기준 → 운영옵션 → 발행), 페이지마다 "저장 후 다음" 반복
- newcourse = 한 화면 통합 (커리큘럼 관리 + 통계 헤더 + 이수조건 인라인 + 코스 퍼블리싱)

## 차용 결정 (AI 제외)

### Phase 1 — 라이브러리에서 가져오기 (우선)

- 위치: 커리큘럼 관리 화면의 "+ 커리큘럼 추가" 클릭 후 드롭다운에 "라이브러리에서 선택" 옵션 추가
- 자산 단위: 영상·시험·과제·설문·자료 5종 (오프라인 강의는 별도)
- 운영팀 우선순위 최고. 매번 같은 오리엔테이션·평가·자료 재업로드 부담 해소

### Phase 2 — 수료기준 비율 슬라이더

- 위치: 헤더 우측 "필수 5" 옆에 "수료비율" 칩 추가. 클릭 시 모달
- 모달 구성: 시험 / 진도율 / 과제 / 설문 → 슬라이더로 비율 합산 100%
- 현재는 항목별 "필수" 체크만 있고 가중치 없음
- Phase 1 미반영 — 이수 조건 인라인 표시로 단위별 판정은 가능

### 후속 점검 태스크

1. **항목 유형 6종 매핑** — 디디쌤은 영상·라이브·오프라인·시험·과제·설문. newcourse 빌더에 라이브·과제·오프라인 항목 유형이 있는지 코드 확인.
2. **단일 vs 과정 클래스** — 디디쌤은 빌더 진입 시점에 분기. newcourse는 "코스 유형 3종(콘텐츠/오프라인/블렌디드)" 분기로 이미 해결됨 ([[project_newcourse_course_types]]).
