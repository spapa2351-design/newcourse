# 2026-05-19 디디쌤 vs newcourse - 커리큘럼 구조 비교 (정정)

질문: 디디쌤이랑 newcourse 커리큘럼 구조가 같은가?

답: **다름. 디디쌤은 재귀 구조, newcourse는 3단 고정.**

## 정정 사항 (2026-05-19 후속 점검)

앞 보고 "디디쌤 = 과정 클래스 L1 / 클래스 L2 / 커리큘럼 L3 3단"은 **부정확**.
디디쌤 가이드 원문(line 200~204) 재확인:

> 커리큘럼 구성 방식에 따라 두 가지 유형으로 나눌 수 있어요.
> • 단일 클래스 — 강의와 시험, 과제 등을 추가하여 자유롭게 구성
> • 과정 클래스 — 기존에 만들어둔 클래스를 조합해서 구성

→ "과정 클래스"는 **별도 상위 layer가 아니라 클래스 만들기 단계의 구성 방식 선택**.
→ 클래스가 다른 클래스를 자식으로 둘 수 있는 **재귀(self-reference) 구조**.

## 정확한 계층

### 디디쌤
- **클래스** (독립 판매·운영 단위)
  - 구성 방식: 단일 클래스 OR 과정 클래스 선택
    - 단일 클래스 = **커리큘럼(항목)** 6종 직접 추가 — 영상·라이브·오프라인·시험·과제·설문
    - 과정 클래스 = **다른 클래스를 자식으로 조합**
- 깊이 고정 X. 클래스가 클래스를 포함하는 재귀 구조.

### newcourse
- **코스 → 커리큘럼(그룹) → 항목(에셋)** 3단 고정
- 코스 묶음/패키지 개념 빌더에 없음

## 용어 충돌 (변함없음)

| 디디쌤 용어 | newcourse 매핑 |
|---|---|
| 클래스 | 코스 (독립 판매·운영 단위) |
| 커리큘럼 | 항목/에셋 (최하위 학습 자산) |
| 과정 클래스 | (없음 — 패키지/번들 개념 미구현) |

## 패키지 코스 구현 옵션 (newcourse 후속 안건)

- **(A) 학습 패스/번들 계층 추가** — 코스 위에 패키지 layer 한 단 추가
  - 장점: 명시적 구조, 권한·결제 단위 분리 용이
  - 단점: 3.5단 깊이로 복잡, 코스/패키지 이중 운영

- **(B) 항목 유형 "다른 코스 참조" 추가** — 디디쌤 스타일 재귀
  - 장점: 기존 3단 구조 유지, 자유로운 조합
  - 단점: 재귀 깊이 제한 정책 필요, 진도율·이수 집계 복잡

## 차용 영향

- **라이브러리에서 가져오기** = 항목 단위 매핑, 구조 차이와 무관 → 이미 구현됨
- **수료기준 비율 슬라이더** = 그룹/항목 가중치 정책 결정 선행 필요
- **패키지 코스** = (A) vs (B) 별도 의사결정 필요. Phase 1 범위 외.
