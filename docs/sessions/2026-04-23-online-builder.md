## 1. admin1.html 변경사항

### 1-1. Phase 3 블렌디드 예고 섹션 삭제
- 수료 조건 섹션 하단의 "블렌디드 코스(Phase 3) 지원 예정 조건" 블록 제거
- 개발 전 단계이므로 UI에서 제거

### 1-2. 수료 조건 섹션 — 실질적 설정 기능 추가

**기존**: 단순 표시용, 설정 불가  
**변경 후**: 최소 출석 기준을 관리자가 설정 가능

```js
const [completionRules, setCompletionRules] = useState([
  { id: 'attendance', slot_type: 'attendance', label: '오프라인 출석', required: true, fixed: true,
    condition: { type: 'any_status', minType: 'all', minCount: 1 } }
]);
```

| 라디오 옵션 | 동작 |
|------------|------|
| 전체 회차 출석 필수 | `minType: 'all'` |
| 최소 N회 이상 | `minType: 'min', minCount: N` |

### 1-3. 기타 제거
- 섹션 헤더의 "Phase 3 블렌디드 확장 가능" 배지 제거
- 조건 카드의 "Phase 1 고정" 배지 제거
- 섹션 부제: `아래 조건을 **모두 충족**해야 수료 처리됩니다.`

## 2. admin-online.html 신규 생성

**경로**: `plan-main/offline/admin-online.html`  
**스택**: React 18 (Babel standalone) + Tailwind CDN + Lucide 0.292.0 — 빌드 없음, 단일 HTML

### 2-1. 레이아웃 규격 (admin1.html 동일)

| 항목 | 값 |
|------|-----|
| 메인 섹션 폭 | `w-[800px] mx-auto` |
| 우측 사이드바 폭 | `w-[500px]` |
| 바디 | `h-screen flex flex-col overflow-hidden` |
| 스크롤 | `main`만 `overflow-y-auto`, body `overflow-y-auto` |

### 2-2. 4개 섹션 (좌측 메인)

| # | 섹션명 | 온라인 특화 항목 |
|---|--------|----------------|
| 1 | 코스 기본 정보 | 난이도(입문/중급/고급), 예상 학습 시간, 수료증 발급 토글 — 장소/주소 없음 |
| 2 | 수강 신청 설정 | 오프라인과 동일 (신청기간, 자율/강제, 취소정책) |
| 3 | 수강 기간 설정 | 고정기간 / 신청 후 N일 / 무제한 3가지 모드 |
| 4 | 수료 조건 설정 | 필수 콘텐츠 전체 이수(고정) + 선택 콘텐츠 N개 이상(토글 추가) |

### 2-3. 우측 사이드바: 커리큘럼 관리 (슬롯 → 인덱스 구조)

**정책**: "슬롯"은 백엔드 데이터 모델 용어. UI에는 번호가 붙은 커리큘럼 항목으로 표시.

```
커리큘럼 관리
[+ 영상] [+ 시험] [+ 설문] [+ 아티클]

커리큘럼 (5개 항목)          필수 4 · 선택 1
┌──────────────────────────────────────────┐
│ ≡  1  🎬  오리엔테이션 — 코스 소개   [필수] │
│ ≡  2  📖  사전 읽기 자료              [선택] │
│ ≡  3  🎬  1강. 핵심 개념 이해        [필수] │
│ ≡  4  📋  중간 점검 퀴즈             [필수] │
│ ≡  5  📊  만족도 설문                [필수] │
└──────────────────────────────────────────┘
```

- 드래그앤드롭 순서 변경
- 제목 인라인 편집 (클릭 즉시)
- hover 시: 이수 조건 텍스트 + ✓(필수전환) / ⚙(이수조건) / 🗑(삭제) 버튼

### 2-4. 슬롯 유형별 이수 조건

| 유형 | 기본 조건 | 설정 가능 항목 |
|------|-----------|--------------|
| 영상 | 시청률 90% 이상 | 시청률 슬라이더 (50~100%) |
| 시험 | 60점 이상 / 20문항 | 합격 점수, 총 문항 수 |
| 설문 | 전 문항 응답 후 제출 | 고정 (변경 불가) |
| 아티클 | 전체 열람 완료 | 고정 (변경 불가) |

### 2-5. 퍼블리싱 검증 체크리스트

| 항목 | 조건 |
|------|------|
| 코스명 입력 | `title.trim()` 존재 |
| 담당 강사 지정 | `instructor.trim()` 존재 |
| 신청 기간 설정 | start + end 모두 입력 |
| 수강 기간 설정 | 유형에 맞는 값 입력 |
| 커리큘럼 최소 1개 | `slots.length > 0` |
| 필수 콘텐츠 최소 1개 | `reqSlots.length > 0` |

## 3. 설계 결정사항

### 3-1. 수료 조건 섹션 위치 — 좌측 메인 유지
- 수료 조건은 코스 레벨 정책 → 좌측 메인 섹션 유지
- 우측 사이드바는 슬롯(항목) 레벨 편집 전용
- 대정책(plan1.html) Slot 귀속 원칙과 일치: "슬롯 Pass ≠ 코스 수료"

### 3-2. 수료 조건 Phase 수렴 패턴
- 오프라인: `오프라인 출석` 카드 (최소 N회 설정 가능)
- 온라인: `필수 콘텐츠 전체 이수` 카드 (+ 선택 콘텐츠 N개 옵션)
- 블렌디드(Phase 3): 두 카드가 AND 조건으로 결합되는 구조 예정
- 공통: `completionRules` 배열 + AND 로직 패턴 유지

### 3-3. 슬롯 vs 커리큘럼 항목 용어 분리
- **UI**: "커리큘럼", "항목", 번호 인덱스
- **코드**: `slots` state, `SLOT_TYPES`, `slot.id/type/cond` (백엔드 연동 대비)
