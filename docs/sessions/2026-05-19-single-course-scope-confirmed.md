# 2026-05-19 Phase 1 단일 코스 스코프 확정

## 결정

newcourse Phase 1 빌더 = **단일 코스만**. 패키지·번들·재귀 구조 일체 제외.

## 범위 외 확정 (Phase 1 제외)

- 패키지 코스 / 묶음 코스 / 코스 번들
- 학습 패스 (Learning Path)
- 코스 재귀 (코스 안에 다른 코스 포함)
- 디디쌤 "과정 클래스" 패턴 차용
- 코스 위 상위 계층 추가

## 단일 코스 안 디디쌤 차용 재정렬

| 우선순위 | 항목 | 위치 | 비고 |
|---|---|---|---|
| 1 | **수료증 발급 on/off (코스 단위)** | 사이드바 헤더 옆 칩/모달 | 운영팀 최빈도 요청 |
| 2 | **항목 유형 "과제" 추가** | `SLOT_TYPES` 한 종류 추가 | 디디쌤 6종 vs newcourse 4종 갭 |
| 3 | **라이브 항목** | 항목 추가 모달에서 조건부 | 블렌디드 코스일 때만 |
| 4 | **수료기준 비율 슬라이더** | 사이드바 헤더 옆 칩/모달 | 그룹/항목 가중치 정책 결정 선행 |

## 미적용 사유

- 패키지/번들/재귀 = 데이터 모델·결제 단위·진도율 집계 복잡도 폭증. Phase 1은 단일 코스만 단단히.
- 운영팀이 패키지 코스 요청 시 → Phase 2 안건으로 분리, 단일 코스 빌더 마무리 우선.

## 후속 작업

- 빌더 코드에 단일 코스 가정이 일관되게 들어가 있는지 점검 (`chapters` 평면 배열, 코스 자기참조 필드 없음 확인)
- 사이드바 헤더에 "수료증 발급" 칩 추가 (1순위 디디쌤 차용)
- 항목 추가 모달 `SLOT_TYPES`에 "과제" 종류 추가 (2순위)
