# 출석률 계산 & 차수·회차 변경 정책 Q&A

> 2026-05-06 · 개발자 ↔ 기획자 문답 기록

---

## Q1. 회차 변경/추가/삭제 시 학습자 출석률 재계산 정책은?

> **개발자**: 출석 회차를 변경/추가/삭제 하게 될 때 학습자 출석률 재계산 정책을 정해야 해요. 출석률 분모가 뭐냐가 핵심입니다.
>
> - **Option A** — 차수 전체 회차 수 (미래 포함). 회차 추가 즉시 분모 증가 → 기존 수강생 출석률 하락. ELIGIBLE이 강등되는 상황 발생.
> - **Option B** — 마감된 회차 수만. 미래 회차는 분모에 포함 안 됨. 회차 추가는 snapshot에 즉각 영향 없음, re-fanout 불필요. 마감된 회차 삭제만 re-fanout 필요.

**기획자**: **Option B로 갑니다.** 결론부터 드리고 정리할게요.

### 왜 B인가

1. **학습자 공정성** — 아직 열리지도 않은 회차 때문에 rate가 떨어지는 건 부당해요. 회차 추가는 보통 강사 사정·보강처럼 학습자 책임이 아닌 사유로 일어납니다.
2. **운영 가능성** — 차수·회차 추가는 운영 일상이에요. 매번 re-fanout이 트리거되면 시스템이 못 버팁니다. 운영팀 부담도 폭증.
3. **수료 시점 결과 동일** — 코스 종료 시점에는 A·B 결과가 어차피 같아요. 차이는 진행 중 표시에서만 납니다.
4. **법정의무 75% 정의 일치** — 법정 기준은 "최종 출석률"이라 진행 중 표시는 "현재까지의 출석률"이 자연스러움.

### 비유로 설명

| | A 방식 | B 방식 |
|--|--|--|
| 비유 | 시험 진행 중인데 안 푼 문제까지 0점 처리 | 지금까지 푼 문제 중 몇 점 맞았는지 |
| 학습자 인상 | "20%? 망했네…" | "지금까지 67%. 남은 회차 잘 챙기면 회복 가능" |

10회차 코스, 3회차 마감, 2회 출석한 학습자 기준:
- A: 2/10 = **20%** (미래 7회를 다 결석으로 가정)
- B: 2/3 = **67%** (지금까지 마감된 3회만 분모)

### 보완 정책 5가지

| # | 정책 | 이유 |
|--|--|--|
| 1 | 학습자 앱에 "마감된 N회차 중 M회 출석 (X%)" 형식으로 분자·분모 모두 노출 | "왜 이 숫자?" 안 묻게 |
| 2 | 미래 회차 추가 시 학습자에게 푸시·메일 알림 ("일정 1개 추가됨") | 출석률은 안 변해도 일정 변경 자체는 통지 |
| 3 | 수료/미수료 라벨은 차수 종료 후에만 노출. 진행 중에는 출석률 숫자만 표시 | UI 부담으로 "현재까지" 별도 라벨 제거. 차수 상태 배지(진행중/완료)가 finality 시그널 대체 |
| 4 | 마감된 회차 삭제는 admin 이중 확인 + audit log + re-fanout 트리거. 영향 학습자 수 dialog에 미리 표시 | 드물지만 영향 큼 |
| 5 | 회차 추가·삭제 모두 audit log에 [시점·작업자·영향 인원] 기록 | 클레임 추적 가능하게 |

## Q2. 차수가 늘고 회차도 늘었는데, 이미 수료한 학습자는 어떻게 되나요?

> **개발자**: 2차수였던 코스가 1차수가 늘어났고, 회차도 10개 추가됐어요. 학습자는 2차수에 등록해서 다 출석해 수료 표시까지 떴는데, 이렇게 되면 사용자 결과는 어떻게 됩니까?

**기획자**: 이건 정책으로 못 박을 케이스예요. 단위 합의부터 시작합시다.

### 단위 합의: 출석률은 차수(Session) 단위

- **분모** = 학습자가 등록된 **그 차수**의 마감된 회차 수
- **분자** = 그중 출석/지각/조퇴한 회차 수

학습자는 어차피 한 차수에만 등록되므로, **다른 차수에 뭐가 일어나든 본인 출석률·수료에는 영향 없습니다.**

### 시나리오 매트릭스

| 어떤 변화? | 2차수 등록·완료자(수료 떴음)에게 결과 |
|--|--|
| 새 차수(3차수) 추가 | **영향 없음**. 수료 유지 |
| 새 차수(3차수)에 회차 10개 추가 | **영향 없음**. 수료 유지 |
| 본인 차수(2차수)에 회차 10개 추가 — 이미 종료(마감) 상태 | 시스템에서 **회차 추가 자체를 차단**. 사용자 영향 없음, 수료 유지 |
| 본인 차수(2차수)에 회차 10개 추가 — 진행 중 | 분모 증가 → 임시 수료가 풀려 "진행 중"으로 변경. 학습자 알림 발송 |

질문 시나리오는 첫 두 케이스(차수 추가 + 그 차수에 회차 추가)이므로, **사용자는 영향 없음. 수료 표시 그대로.**

### 정책 약속 3개

1. **차수가 종료(closed)되면 그 차수의 출석률·수료는 immutable. 사후 변경 불가.**
2. **종료된 차수에는 회차 추가/삭제가 시스템에서 차단된다.** UI 버튼도 비활성화.
3. **진행 중 차수에 회차 추가 시 학습자 알림 + "회사 사정으로 추가됨" 안내 자동 발송.**

핵심 한 줄: **"내 차수에서 내 약속(75% 출석)을 지켰으면 끝."**

## Q3. 그래도 운영 들어가면 차수·회차 추가가 자주 일어나지 않나요?

> **개발자**: 솔직히 코스 오픈하고 나서도 계속 차수·회차를 추가하는 운영 케이스가 있긴 하지 않을까요? 많지는 않겠지만.

**기획자**: 솔직히 — **꽤 자주 있습니다.** 운영 일상이에요.

### 차수 추가가 흔히 일어나는 케이스

| 코스 | 패턴 | 빈도 |
|--|--|--|
| 매니저 코칭 스킬 워크숍 | 정원 빨리 마감 → 분기마다 차수 추가 | 분기 1회 |
| 데이터 분석 입문 | 5분 만에 마감 → 매월 차수 추가 | 월 1회 |
| 김난도 트렌드 특강 | 외부 강사 일정 추가 받아 차수 추가 | 비정기 |
| 신입 OT | 추가 입사자 모아 보충 차수 운영 | 반기 1~2회 |
| 조직개편/신규 본부 | 그 본부용 차수 추가 | 연 1~2회 |
| 법정의무 미수료자 사이클 | 75% 못 채운 사람 모아 추가 차수 | 분기 1회 |

### 회차 추가가 일어나는 케이스

| 케이스 | 발생 조건 | 빈도 |
|--|--|--|
| 보강 회차 | 강사 사정·공휴일·재해로 결강 | 분기 1~2회 |
| 연장 운영 | 진도 못 빼서 1주 연장 | 분기 1회 |
| Q&A·심화 세션 추가 | 학습자 요청 | 비정기 |
| 법령 개정 시수 증가 | 의무교육 시수 늘어남 | 연 0~1회 |
| 강의장 사정으로 분할 | 한 회차를 두 회차로 쪼갬 | 비정기 |

### 그래서 추가 운영 가이드 3개

**가이드 1. "차수 추가 vs 새 코스 개설" 판단 기준**

| 상황 | 차수 추가 | 새 코스 개설 |
|--|--|--|
| 같은 강사·같은 커리큘럼 | ✅ | |
| 강사·커리큘럼 다름 | | ✅ |
| 시즌(분기·연도) 바뀜 | | ✅ |
| 미수료자 재이수 | | ✅ (이력 분리 위해) |

**가이드 2. "회차 추가 vs 회차 일정 변경" 판단 기준**

- 기존 회차의 날짜·시간만 바뀜 → **회차 일정 변경** (새 회차 추가 X)
- 새 날짜에 추가 수업 → **회차 추가**

**가이드 3. 회차 추가 시 학습자 안내 템플릿 표준화**

> "[코스명]의 일정이 변경되었습니다. [날짜]에 [N회차]가 추가되었어요. 회사 사정으로 인한 변경이며, 본인의 기존 출석률은 영향받지 않습니다. 추가된 회차는 별도 마감 시점에 출석률 분모에 반영됩니다."

운영자가 매번 안내 문구 새로 짜지 않게 시스템에서 자동 발송.

## 핵심 요약

1. **출석률 분모 = 마감된 회차 수만** (Option B)
2. **계산 단위 = 차수(Session)**. 학습자가 등록된 차수 안에서만 계산
3. **차수 종료 시점부터 결과는 immutable**, 회차 추가·삭제 차단
4. **다른 차수 변경은 본인에게 영향 없음**
5. **차수·회차 추가는 운영 일상**이므로 정책이 단순할수록 운영 가능
6. **학습자 앱 표시는 "마감된 N회차 중 M회 출석 (X%)"** 형식만 사용. 별도 라벨 없이 차수 상태 배지(진행중/완료)와 함께 노출
