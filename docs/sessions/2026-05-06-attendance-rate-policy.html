출석률 계산 & 차수·회차 변경 정책

개발자 ↔ 기획자 문답 기록 — 분모 정의 · immutable 차수 · 운영 가이드

👨‍💻 개발자
출석 회차를 변경/추가/삭제 하게 될 때 학습자 출석률 재계산 정책이 필요합니다. 분모가 차수 전체 회차 수(미래 포함)면 회차 추가 즉시 분모 증가로 ELIGIBLE이 강등될 수 있고, 마감된 회차 수만이면 미래 회차 추가/삭제는 snapshot 영향이 없어요. Option B가 맞다면 마감된 회차 삭제 예외만 대응하면 됩니다.
🙋 기획자

Option B로 갑니다. 분모는 마감된 회차 수만 카운트.

왜 B인가

근거설명
학습자 공정성안 열린 회차 때문에 rate가 떨어지는 건 부당. 회차 추가는 보통 학습자 책임이 아닌 사유로 일어남
운영 가능성차수·회차 추가는 일상. 매번 re-fanout 트리거되면 시스템 못 버팀
수료 시점 동일코스 종료 시점엔 A·B 결과가 같음. 차이는 진행 중 표시에서만
법정의무 정의 일치75% 법정 기준은 "최종 출석률". 진행 중 표시는 "현재까지의 출석률"이 자연스러움

비유로 보면

A는 시험 진행 중인데 아직 안 푼 문제까지 0점 처리한 점수. B는 지금까지 푼 문제 중에서 몇 점 맞았는지 보여주는 점수.

10회차 코스, 3회차 마감, 2회 출석한 학습자:

계산표시 출석률학습자 인상
A2 ÷ 1020%"망했네…"
B2 ÷ 367%"남은 회차 챙기면 회복 가능"

보완 정책 5가지

#정책
1학습자 앱에 마감된 N회차 중 M회 출석 (X%) 형식으로 분자·분모 모두 노출
2미래 회차 추가 시 학습자 푸시·메일 알림 ("일정 1개 추가됨")
3수료/미수료 라벨은 차수 종료 후에만 노출 — 진행 중에는 출석률 숫자만 표시 2026-05-06 수정: UI 부담으로 "현재까지" 별도 라벨 제거. 차수 상태 배지가 finality 시그널 대체
4마감된 회차 삭제는 admin 이중 확인 + audit log + re-fanout 트리거. 영향 학습자 수 dialog에 미리 표시
5회차 추가·삭제 모두 audit log에 [시점·작업자·영향 인원] 기록
👨‍💻 개발자
2차수였던 코스가 1차수가 늘어났고, 회차 10개도 추가됐어요. 학습자는 2차수에 등록해서 다 출석 → 수료 표시까지 떴는데, 이렇게 되면 사용자 결과는 어떻게 됩니까?
🙋 기획자

단위 합의: 출석률은 차수(Session) 단위

분모 = 학습자가 등록된 차수의 마감된 회차 수
분자 = 그중 출석/지각/조퇴 회차 수

학습자는 한 차수에만 등록되므로, 다른 차수에 뭐가 일어나든 본인 출석률·수료에 영향 없음.

시나리오 매트릭스

어떤 변화?2차수 등록·수료자에게 결과
새 차수(3차수) 추가영향 없음 수료 유지
새 차수(3차수)에 회차 10개 추가영향 없음 수료 유지
본인 차수(2차수)에 회차 10개 추가 — 이미 종료(마감)시스템에서 차단 수료 유지
본인 차수(2차수)에 회차 10개 추가 — 진행 중분모 증가 임시 수료 풀림 → "진행 중"
질문 시나리오는 첫 두 케이스에 해당 → 사용자는 영향 없음. 수료 표시 그대로.

정책 약속 3개

#약속
1차수가 종료(closed)되면 그 차수의 출석률·수료는 immutable. 사후 변경 불가
2종료된 차수에는 회차 추가/삭제가 시스템에서 차단. UI 버튼 비활성화
3진행 중 차수에 회차 추가 시 학습자 알림 + "회사 사정으로 추가됨" 안내 자동 발송
한 줄 요약 — "내 차수에서 내 약속(75% 출석)을 지켰으면 끝."
👨‍💻 개발자
솔직히 코스 오픈하고 나서도 계속 차수·회차를 추가하는 운영 케이스가 있긴 하지 않을까요? 많지는 않겠지만.
🙋 기획자

솔직히 — 꽤 자주 있습니다. 운영 일상이에요.

가이드 2. "회차 추가 vs 일정 변경" 판단 기준

  • 기존 회차의 날짜·시간만 바뀜 → 회차 일정 변경 (새 회차 추가 X)
  • 새 날짜에 추가 수업 → 회차 추가

가이드 3. 회차 추가 시 학습자 안내 템플릿

[코스명]의 일정이 변경되었습니다.
[날짜]에 [N회차]가 추가되었어요.
회사 사정으로 인한 변경이며, 본인의 기존 출석률은 영향받지 않습니다.
추가된 회차는 별도 마감 시점에 출석률 분모에 반영됩니다.

시스템에서 자동 발송 — 운영자가 매번 안내 문구 새로 짜지 않게.

핵심 요약
1.출석률 분모 = 마감된 회차 수만 (Option B)
2.계산 단위 = 차수(Session). 학습자가 등록된 차수 안에서만 계산
3.차수 종료 시점부터 결과는 immutable, 회차 추가·삭제 차단
4.다른 차수 변경은 본인에게 영향 없음
5.차수·회차 추가는 운영 일상이므로 정책이 단순할수록 운영 가능
6.학습자 앱 표시: 마감된 N회차 중 M회 출석 (X%) 형식만 사용 (별도 라벨 X). 차수 상태 배지로 진행/완료 구분
