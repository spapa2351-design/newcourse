# 오프라인 코스 수정 공유 (2026-06-01) — 개발·기획팀

오늘 오프라인 코스(관리자 빌더 + 학습자 앱)에서 정리한 내용입니다. **수정 전 → 후**로 쉽게 정리했습니다.

> 한 줄 요약: **사용자는 "차수 하나(또는 여러 개) 고르기"만 하고, 수료는 "출석률 %"로 본다. 관리자·사용자 용어와 화면을 통일했다.**

---

## 1. 수강 신청 방식

| 전 | 후 |
|----|----|
| 모드가 흩어져 헷갈림 | **단일 선택(차수 1개) / 다중 조합(여러 차수) / 관리자 지정 등록** 3가지로 정리 |
| "회차 선택형" 위치 모호 | **단일 선택형의 하위 옵션으로 유지** — 차수에 "회차 선택형" 토글 ON 시 학습자가 회차(날짜) 1개 선택. 차수 1개 + 회차 여러 개로 운영(날짜마다 차수를 따로 안 만들어도 됨 → 운영 입력 절감) |

## 2. 다중 조합형 — 일정 겹침

| 전 | 후 |
|----|----|
| 여러 차수를 골라도 **날짜가 겹쳐도 그냥 신청**됨 | 빌더에서 **차수 일정이 겹치면 다중 조합형 비활성 + 발행 차단** (학습자는 동시에 두 곳 출석 불가) |
| — | 판정은 **날짜+시간만**(장소·교실 무관). 학습자단은 관리자가 보장하므로 별도 검사 없음 |

## 3. 수료 조건

| 전 | 후 |
|----|----|
| "최소 N**회** / 총 M회" 횟수 방식, 분모가 전체 차수 합 → 반마다 회차 수 다르면 **깨짐** | **"출석률 75% 이상이면 수료"** — 본인이 들은 차수의 회차 기준. 지각=출석 인정 |
| 문구(회)와 데이터(차수)가 불일치 | 공식 문서(`offline-course.html` 4.4)와 **일치** |

## 4. 용어 통일 (관리자 = 사용자 같은 단어)

| "관리자가 **강제 할당**" | **"관리자 지정 등록"** |

- 학습자: `user/course-detail.html` (3종 케이스·Toss 디자인·케이스 배너·Material Symbols), `course-detail-newin.html`(라벨)
- 문서: `01-plan`/`02-design` offline-builder, `00-standard/offline-terminology.md`, sessions 3건

## 개발 참고 포인트

- 수료 데이터: `condition.minRate`(출석률 %) — 기존 minType/minCount(회차 횟수) 제거.
- 신청 모드 값: `single` / `multiple` / `assigned`. 다중 수료 = **선택한 차수들의 회차 합산 출석률** (백엔드 산식 확정 필요).
- 빌더 겹침 검사: `overlapPairs`/`hasOptionOverlap`(날짜+시간 교차).
- `sessionSelect`(회차 선택형): **유지** — 차수 단위 토글. ON 시 학습자가 회차(날짜) 1개 선택.

## 회차 선택형 — 폐지 검토했으나 "유지" 재확정 (중요)
- 오전엔 "폐지(날짜별 차수로 분리)"로 갔으나, **운영 입력 부담**(날짜 10개=차수 10개를 일일이 생성·정원관리) 때문에 **plan에서 "유지"로 재확정**.
- 최종: **회차 선택형 = 단일 선택형 하위 옵션(차수 1개 + 회차 여러 개, 토글 1개).** 관리자는 차수 하나만 만들고 날짜(회차)만 추가.
