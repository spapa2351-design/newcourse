오프라인 코스란?

오프라인 LMS는 강의실 교육을 웹으로 관리하는 시스템입니다. 수강 신청부터 QR 출석, 수료 처리까지 전 과정을 하나의 앱에서 처리할 수 있습니다.

카테고리

분야별로 코스를 탐색하는 화면입니다. 상단 필터로 온라인/오프라인을 구분해 볼 수 있으며, 오프라인 카테고리를 선택하면 해당 코스 목록으로 바로 이동합니다.

① 기본 화면 — 전체 카테고리
기본값은 "전체"로, 온라인 코스 카테고리와 오프라인 코스 카테고리가 섹션별로 나뉘어 표시됩니다.
전체 카테고리
코스 · PLUS 코스 · 오프라인 코스 섹션으로 구분
② "오프라인만" 필터 → 오프라인 카테고리만 표시
상단 필터에서 오프라인만 버튼을 탭하면 온라인 카테고리가 숨겨지고 오프라인 카테고리 6종만 보입니다.
"오프라인만" 필터 활성

수강 모드 비교
자율 단일
자율 다중
강제배정
수강 신청 방법은 모드별로 다릅니다. 각 모드의 상세 안내는 "수강 신청" 섹션을 참고하세요.
차수(반) 선택 → 회차별 일정 확인
커리큘럼 목록에서 차수(반)를 탭하면 해당 차수 안의 회차별 날짜·시간·장소가 펼쳐집니다. 모달에서도 동일하게 차수 선택 시 회차 목록이 표시됩니다.

신청 기간 & 나의 수강 상태

수강 신청 기간과 학습자의 수강 상태에 따라 코스 상세 화면이 다르게 표시됩니다. 아래 7가지 케이스를 확인하세요.

1. 신청 기간별 상태 (미신청자 기준)
운영자가 설정한 수강 신청 기간에 따라 버튼과 배지가 달라집니다.
신청 기간 중
커버 배지 "신청 가능"
"수강 신청하기" 버튼 활성
신청 오픈 전
커버 배지 "오픈 예정"
오픈 일시 표시, 버튼 비활성
신청 마감
커버 배지 "신청 마감"
"신청이 마감되었습니다" 버튼 비활성
2. 나의 수강 상태별 화면
이미 신청하거나 완료한 학습자에게는 신청 버튼 대신 수강 상태 배너가 표시됩니다.
수강중
초록색 "수강 중" 배너
"내 수강 상세 보기" 버튼으로 전환
수강 완료
회색 "수강이 완료된 코스" 배너
버튼 비활성 (출석률 표시)
3. 상태별 요약표
상태 커버 배지 하단 버튼 배너
신청 오픈 전 오픈 예정 ⏳ 오픈 일시 안내 (비활성) 없음
신청 기간 중 신청 가능 수강 신청하기 → (활성) 없음
신청 마감 신청 마감 신청이 마감되었습니다 (비활성) 없음
수강중 수강중 내 수강 상세 보기 → (활성) 초록색 수강중 배너
수강 완료 수강 완료 수강 완료 · 출석률 100% (비활성) 🏆 회색 완료 배너

관리자 화면 개요

오프라인 LMS 관리자 화면은 코스 개설부터 수강생 관리, 출석·수료 처리까지 전 과정을 운영하는 시스템입니다. 5개 주요 화면으로 구성됩니다.

화면 구성 5종
📋
코스 목록 (offline-list.html)
전체 오프라인 코스 목록. 상태 필터, 분류 사이드바, 차수 확장 보기. 코스 등록/수강관리 진입.
🛠️
코스 등록 빌더 (offline-builder.html)
코스 기본정보, 수강 신청 설정, 수료 조건을 좌측 폼에서 입력. 우측 사이드바에서 차수·회차 관리.
📊
코스 운영 현황 (offline-view.html)
운영 중 코스의 대시보드, 수강생 관리, 출석·수료 처리, QR 라이브 모드까지 통합 운영 화면.
👤
수강생 등록 (course-learners.html)
수강생 개별 등록 및 일괄 등록. 차수 배정, 출석 탭 관리.

Phase 1에선 미확인 인원을 운영자가 직접 처리합니다. 자동 결석 전환은 Phase 2 이월.
학습 시간
학습 시간 = 출석한 회차의 수업 시간(종료−시작) 합산. 조퇴·지각도 그 회차 전체 시간 인정, 결석·미확인은 0. 표시: 회차 테이블 "학습 시간" 칸 · 차수 헤더 합계 · 좌측 코스 정보 · 상단 "평균 학습시간" 카드.
회차 설정에서 자동 계산이 기본, 관리자가 "직접 입력" 체크 시 회차별 수정 가능(수정 시 "수정됨" 배지). Phase 1은 표시 전용 — 수료 기준은 출석률 75% 그대로, 국비교육 시수 기준은 Phase 3 분리. 개인별 학습시간은 출석률과 중복이라 명단에선 생략.
회차 종료, 한눈에 이해하기

학습자가 자주 물어요. "오늘 못 가면 저 한 회차 까이는 거예요?" 답을 출석부 비유로 풀어 드릴게요.

🎯 한 줄로 말하면
회차 종료는 수업 종료 시각이 지나는 시점이에요.
그 시점부터 그 수업은 학습자의 출석 점수에 한 칸으로 들어갑니다.
종료 전엔 점수에 안 들어가요. 별도 "마감 버튼"은 없어요 — 시간이 결정합니다.
🪧 4번 수업 교육 예시
시점 학습자 화면 의미
1·2번째 수업 종료 전 "진행 중" 아직 종료 안 됨 → 점수에 안 들어감
1번 수업 종료 후 1칸 중 출석한 비율 출석부 1칸 채워짐
2번 수업 종료 후 2칸 중 출석한 비율 출석부 2칸 채워짐
4번 수업 모두 종료 최종 출석률 (4칸 기준) 75% 이상이면 수료 대상
👤 학습자에게 일어나는 일
① 출석부 칸이 1개 추가돼요
② 출석/지각/결석 결과가 바로 보여요
③ 미확인이면 "출석 확인 중" 표시
※ Phase 1은 학습자한테 푸시·이메일이 안 나가요. 학습자가 앱·웹에 직접 들어와서 확인. 알림은 Phase 2에서 도입.
📊 운영자 화면에서 일어나는 일
① 출석률 계산이 시작돼요
② 차수 카드 출석률 막대바 즉시 업데이트
③ "출석 확인 중" 인원이 있으면 코스 헤더에 배지 표시
⏰ 회차 종료 (시간 기반 자동)

회차는 수업 종료 시각에 자동으로 종료됩니다 (별도 마감 버튼 없음). 미확인 인원은 운영자가 직접 정리합니다 — "미확인 일괄 처리" 도구로 빠르게 처리 가능 (전원 출석/전원 결석/개별).

※ Phase 1엔 자동 결석 처리·운영자 단계별 알림·시스템 잠금 정책 없음. Phase 2 이월.
📢 학습자 알림은 Phase 2에서 도입 Phase 2

Phase 1은 학습자한테 푸시·이메일이 안 나갑니다. 학습자가 앱·웹에 직접 들어와서 자기 출석 결과 확인.

Phase 2 알림 5종 분리 (예정)
  • C1 출석 처리됨 (즉시) — QR 찍자마자 "출석 처리됐어요"
  • C2 출석률 표시 (즉시) — 회차 종료 직후 화면 갱신
  • C3 미확인 처리 중 (즉시·약한 알림) — "운영자 확인 중"
  • C3 결석 확정 — 운영자가 결석 처리 완료 후 발송
  • C4 HR 시스템 연동 — 운영자가 명시한 시점에 발송
✏️ 종료 후에도 출석은 고칠 수 있어요
• 결석 → 출석으로 바꾸는 건 회차 종료 후에도 OK (관리자만)
• 바꾸자마자 출석률이 다시 계산돼요
• "마감 사건"은 개념 자체가 없어요. 수업 종료 시각이 지나면 자동 종료, 그 사실은 변하지 않음
• Phase 2부터: 수료 확정까지 받은 학습자는 출석을 고쳐도 수료 상태가 자동으로 안 바뀝니다
👥 운영자 여러 명이 같이 보고 있을 때
• 라이브 모드를 여러 PC에 동시에 띄우는 건 OK예요. 100명 이상 큰 행사에서는 입장 게이트 2개로 나누고 QR 화면 2대 띄우는 게 정식 권장이에요
• 마감 버튼이 없어 동시 클릭 충돌 자체가 없어요 (시간 기반 자동 종료)
• 실무 팁: 회차마다 "미확인 인원 정리 책임자 1명"을 정해두세요. 운영팀 내부 약속으로 잡아두면 빠릅니다
🚫 수업이 아예 못 열린 경우 — 휴강 Phase 3

강사가 갑자기 못 오거나, 강의장 사고나 천재지변으로 수업 자체가 안 열린 경우. Phase 1·2에서는 운영자가 임의 처리(미확인 일괄 → 출석 또는 결석)로 우회.

Phase 3 추가 예정
  • 회차 상태에 "휴강" 추가
  • 휴강된 수업은 출석부에서 그 칸을 빼버림
  • "4번 수업 중 3번으로 점수 계산"이라고 표시
  • 관리자만 처리 가능, 사유 입력 필수
"보강 강제" 토글 (Phase 3)
  • ON: 법정의무교육 — 보강 안 잡으면 수료 처리 불가
  • OFF: 자율 교육 — 휴강은 빼고 그냥 진행
  • 날짜선택형은 학습자에게 다른 날짜 자동 안내
용어 안내: "회차 종료"로 통일. 학습자가 "회차 차감"이라고 부르는 건 같은 의미예요 (별도 사건이 아니라 수업 종료 시각이 지나는 것). 수업 자체가 못 열린 건 "휴강"입니다 ("회차 취소"는 신청 취소와 헷갈리니 사용 금지).
