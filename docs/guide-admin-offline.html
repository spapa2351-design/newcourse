관리자 화면 개요

오프라인 LMS 관리자 화면은 코스 개설부터 수강생 관리, 출석·수료 처리까지 전 과정을 운영하는 시스템입니다. 5개 주요 화면으로 구성됩니다.

화면 구성 5종
📋
코스 목록 (index.html)
전체 오프라인 코스 목록. 상태 필터, 분류 사이드바, 차수 확장 보기. 코스 등록/수강관리 진입.
🛠️
코스 등록 빌더 (offline-builder.html)
코스 기본정보, 수강 신청 설정, 수료 조건을 좌측 폼에서 입력. 우측 사이드바에서 차수·회차 관리.
📊
코스 운영 현황 (offline-course-view.html)
운영 중 코스의 대시보드, 수강생 관리, 출석·수료 처리, QR 라이브 모드까지 통합 운영 화면.
👤
수강생 등록 (course-learners.html)
수강생 개별 등록 및 일괄 등록. 차수 배정, 출석 탭 관리.

③ 수료 조건

수료 처리를 위한 출석 조건을 설정합니다. 전체 회차 출석 또는 최소 N회 출석 중 선택합니다.

출석 조건 2종
전체 회차 출석
모든 회차에 출석해야 수료 인정. 기본값.
최소 N회 출석
지정한 횟수 이상 출석 시 수료 인정. 지각도 출석 1회로 인정.
지각(late)과 조퇴(early_leave)는 모두 출석 1회로 인정됩니다(코스 유형 무관 공용 기본정책). 결석·미확인만 불인정. 조퇴·결석엔 사유를 기록합니다. 법정의무교육의 시수 strict(조퇴=미인정)는 Phase 2 분기 예정입니다.

출석·수료 탭

차수별 회차 출석 현황을 관리합니다. 미확인 출석 일괄 처리, 수동 상태 변경 기능을 제공합니다. 회차는 수업 종료 시각에 자동 종료되며 (별도 마감 버튼 없음 — 2026-05-06 정책 변경), 미확인 인원은 운영자가 직접 정리합니다.

Phase 1에선 미확인(pending) 인원을 운영자가 직접 정리합니다 ("미확인 일괄 처리" 도구 사용). 자동 결석 전환은 Phase 2 이월.
학습 시간
학습 시간 = 출석한 회차의 수업 시간(종료−시작) 합산. 조퇴·지각도 그 회차 전체 시간 인정, 결석·미확인은 0. 표시 위치: 회차 테이블 "학습 시간" 칸 · 차수 헤더 합계 · 좌측 코스 정보 · 상단 "평균 학습시간" 카드.
회차 설정에서 자동 계산이 기본, 관리자가 "직접 입력" 체크 시 회차별 수정 가능(수정 시 "수정됨" 배지). Phase 1은 표시 전용 — 수료 기준은 출석률 75% 그대로, 국비교육 시수 기준은 Phase 3 분리. 개인별 학습시간은 출석률과 중복이라 명단에선 생략.
