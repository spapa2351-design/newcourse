# 오프라인 코스 등록 — 공식 기획 Plan

---

## Executive Summary

| 관점 | 내용 |
|------|------|
| **Problem** | 고객사가 오프라인 교육을 엑셀 + 수동 운영으로 관리 중. 수료율 집계·미수료자 추적·출석 관리가 사람 손에 의존해 오류·누락 발생 |
| **Solution** | HRD 담당자가 코스를 직접 개설하고, QR 출석·수료 처리·통계까지 시스템에서 자동화 |
| **기능/UX 효과** | 코스 개설 → 차수/회차 설정 → 수강 신청 → QR 출석 → 회차 마감 → 수료 처리 → 통계 확인의 완전한 루프 제공 |
| **Core Value** | "엑셀을 버릴 수 있는 최소 시스템" — 자동화와 데이터 출력이 가능한 수준으로 실제 고객사 납품 가능 |

## 1. 사용자 인텐트 발견 (Intent Discovery)

### 핵심 문제
오프라인 교육 운영이 엑셀 + 수동 방식으로 운영되어 자동화와 데이터 관리가 필요함.

### 주요 사용자

| 사용자 | 역할 | 핵심 니즈 |
|--------|------|-----------|
| **HRD 담당자 (관리자)** | 코스 개설, 차수·회차 설정, 수강생 등록, 출석·수료 관리 | 엑셀 없이 전체 운영 가능한 시스템 |
| **학습자 (수강신청자)** | 코스 조회, 자율 신청, 출석 QR 스캔 | 간편한 신청 및 출석 확인 |
| **강사** | QR 출석 화면 운영, 수강생 명단 확인 | 현장 운영 편의성 |
| **상위 관리자 (경영진)** | 수료율·출석률 통계 확인 | 한 눈에 보이는 리포트 |

### 성공 기준
- HRD 담당자가 코스 개설부터 수료 처리까지 엑셀 없이 처리 가능
- 실제 고객사에 납품하여 오프라인 운영에 투입 가능한 수준
- 차수별 출석률·수료율 데이터가 자동으로 집계됨

## 4. Phase 1 필수 결정 갭

### GAP-01: 대기자 플로우 ✅ 결정 완료

> **결정**: 대기 없음 — 정원 마감 시 다른 차수 유도

- 정원 초과 시 "마감" 배지 표시 + 다른 차수 목록 안내
- 대기자 기능은 Phase 2에서 실제 운영 피드백 후 추가 여부 결정
- **이유**: Phase 1은 엑셀 대체가 목표. 대기자 플로우는 시스템 복잡도를 크게 높임.

---

### GAP-03: 알림 채널 ⚠️ 결정 필요 (오픈 전)

신청 완료·D-day 리마인더 등의 알림을 어떤 채널로 보낼지 미결.

| 옵션 | 구현 난이도 | 비고 |
|------|------------|------|
| 이메일 | 낮음 | 외부 API 연동 없이 가능. 대부분의 고객사 보유 |
| 앱 푸시 | 중간 | 학습자 앱 존재 시 가능 |
| SMS | 높음 | 외부 문자 API 필요, 비용 발생 |

> **권고**: Phase 1에서는 이메일만 지원, 앱 푸시는 Phase 2에서 학습자 앱 연동 시 추가.
> **→ 오픈 전 반드시 확정 필요**

## 5. 아키텍처 결정사항 요약

### 계층 구조 (확정)

```
코스 (무엇을 배우나 — 제목·강사·분류·신청 설정)
 └── 커리큘럼 (어떻게 배우나 — 슬롯 순서/구성)
       └── 슬롯 (개별 학습 단계)
             └── 오프라인 출석 슬롯
                   └── 차수 (운영 회기: 날짜·정원·그룹)
                         └── 회차 (개별 수업일)
```

### 용어 체계 (확정)

| 용어 | 의미 |
|------|------|
| 코스 | 교육 프로그램 단위 |
| 차수 | 오프라인 운영 회기 (구 "패키지") |
| 회차 | 차수 내 개별 수업일 |

### 기술 스택

- React 18 (Babel standalone, ESM CDN) — 빌드 없이 브라우저 직접 실행
- Tailwind CSS (CDN)
- Lucide React 0.292.0
- 단일 HTML 파일 구조

### 출석 상태 5종 (확정)

| 상태 | 키 | 설명 |
|------|-----|------|
| 미확인 | `pending` | 회차 생성 시 기본값 (임시 상태) |
| 출석 | `attended` | QR 스캔 또는 관리자 수동 |
| 지각 | `late` | 시작 시간 이후 QR 스캔 또는 수동 |
| 조퇴 | `early_leave` | 관리자 직접 변경 |
| 결석 | `absent` | 수동 또는 회차 마감 시 일괄 전환 |
