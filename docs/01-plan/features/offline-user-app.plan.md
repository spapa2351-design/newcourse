## Executive Summary

| 관점 | 내용 |
|------|------|
| **Problem** | 오프라인 교육 운영 시스템(관리자단)은 Phase1 완성 상태지만, 학습자가 수강신청·출석확인·수료결과를 확인하는 앱의 공식 기획 문서가 없어 화면 간 상태 전이·정책 예외 처리·관리자단 연동 기준이 불명확함 |
| **Solution** | 이미 구현된 7개 프로토타입 화면을 기반으로 상태 기반 라우팅 매트릭스, QR 결과 메시지 규격, 수강신청 3모드 확인 흐름, 라이브 모드 연동을 공식 명세화 |

| **Core Value** | "관리자가 개설한 코스를 학습자가 신청·출석·수료까지 앱에서 완결" — 관리자단 Phase1과 1:1 대응하는 학습자 앱 명세 |

## 1. 사용자 인텐트 발견

### 핵심 문제

프로토타입은 이미 구현됐지만 공식 명세가 없어:
- 화면 간 라우팅 분기가 개발자 직관에 의존
- QR 출석 결과 메시지가 기술 중심 (현장 혼란 유발)
- 관리자단 정책(변경·취소 기한, 수강 모드)과 학습자 화면의 연결 고리 미명시

### 주요 사용자

| 사용자 | 역할 | 핵심 니즈 |
|--------|------|-----------|
| **학습자** | 코스 조회, 수강신청, QR 출석 | 내 수강 현황을 한 눈에, QR 체크인 간편하게 |
| **HRD 담당자** | 관리자단에서 코스 개설·출석 관리 | 학습자 앱이 관리자 설정과 1:1 반영되어야 함 |
| **현장 강사** | QR 수업 진행 | 학습자 QR 스캔 결과가 정확해야 함 |

### 성공 기준

1. 코스 상태 × 신청 상태 조합에 따른 라우팅이 모두 정의됨
2. QR 결과 5종의 문구·아이콘·색상이 확정됨
3. 수강신청 3모드(단일/다중/강제)의 학습자 화면 흐름이 명세됨
4. 관리자단 Phase1의 정책(변경기한, 취소기한, sessionSelect)이 학습자 화면에 반영됨
5. 라이브 모드(BroadcastChannel) 시나리오가 명세됨

## 3. 상태 기반 라우팅 매트릭스

> 코스 카드를 탭했을 때 어느 화면으로 이동하는지 규칙 정의

### 3.1 코스 상태 정의

| 코스 상태 | 설명 |
|-----------|------|
| `enrollable` | 신청 기간 내, 신청 가능 |
| `not-started` | 신청 완료, 아직 시작 전 |
| `in-progress` | 수강 중 (회차 진행 중) |
| `scheduled` | 개강 예정 (신청 기간 종료, 미시작) |
| `ended` | 수강 종료 (미수료 포함) |
| `completed` | 수료 완료 |

### 3.2 라우팅 규칙

| 학습자 신청 상태 | 코스 상태 | 이동 화면 | 비고 |
|-----------------|-----------|-----------|------|
| 미신청 | enrollable | `course-detail.html` (신청 화면) | |
| 미신청 | scheduled / not-started / in-progress | `course-detail.html` (신청 불가 안내) | 신청 버튼 비활성 |
| 신청완료 | not-started | `course-enrolled.html` | |
| 신청완료 | in-progress | `course-enrolled.html` | QR CTA 노출 |
| 신청완료 | ended | `course-completed.html` | 수료/미수료 분기 |
| 신청완료 | completed | `course-completed.html` | 수료증 다운로드 활성 |
| 강제배정 | 모든 상태 | 동일 규칙 적용 | 관리자 배정이므로 취소 버튼 정책 별도 |

### 3.3 MY 탭 → 상세 화면 라우팅

| MY 카드 상태 | 이동 화면 |
|-------------|-----------|
| 수강예정 / 수강중 | `course-enrolled.html` |

| 히어로 카드 (D-Day 표시) | `course-enrolled.html` |

## 4. 수강신청 3모드 화면 흐름

### 4.1 모드 정의

| 모드 | `enrollMode` 값 | 설명 |
|------|----------------|------|
| 자율 단일선택 | `single` | 커리큘럼 1개 → 차수 1개 선택 |
| 자율 다중조합 | `multi` | 커리큘럼 N개 → 각 1개 차수 선택 |
| 관리자 강제배정 | `assigned` | 학습자가 선택 불가, 관리자가 배정 |
| 회차 선택형 | `sessionSelect` | 차수 내 회차를 학습자가 선택 (법정의무교육) |

### 4.2 단일선택 흐름 (`single`)

```
코스 상세(screen-detail)
  └─ [수강신청] 버튼 탭
      └─ 바텀시트: 차수 목록 (잔여석·기간 표시)
          └─ 차수 선택
              └─ screen-confirm (선택 내역 1줄 요약)
                  └─ [신청 완료] 버튼
                      └─ 성공 모달 → MY 이동
```

### 4.3 다중조합 흐름 (`multi`)

```
코스 상세(screen-detail)
  └─ [수강신청] 버튼 탭
      └─ 바텀시트: 커리큘럼별 차수 선택 (ucal 미니 캘린더)
          └─ 커리큘럼 A: 차수 선택 / 커리큘럼 B: 차수 선택
              └─ screen-confirm
                  ├─ 선택 요약 카드 (커리큘럼별 선택 차수 목록)
                  └─ [신청 완료] 버튼 → 성공 모달
```

**선택 요약 카드 구조**:
```
선택하신 반
  [커리큘럼 A] 1기 — 2026.05.01 ~ 05.10 (매주 수)
  [커리큘럼 B] 3기 — 2026.05.15 ~ 05.20 (매주 목)
```

### 4.4 강제배정 흐름 (`assigned`)

```
코스 상세(screen-detail)
  └─ 신청 버튼 없음 (배정 안내 문구 표시)
  └─ "관리자에 의해 배정된 코스입니다"
  └─ 배정된 차수 정보 표시
```

- 취소 버튼: 정책에 따라 노출/비노출 (관리자 설정값 우선)

### 4.5 회차 선택형 흐름 (`sessionSelect`)

```
코스 상세
  └─ [수강신청] 버튼
      └─ 바텀시트: 회차 목록 (날짜·시간·장소·잔여석)
          └─ 회차 1개 선택
              └─ screen-confirm
                  └─ [신청 완료]
```

## 5. 변경·취소 정책 화면 반영

### 5.1 정책 상태 → UI 매핑

| 정책 상태 | UI 처리 |
|-----------|---------|
| 기한 내 | 일정변경 버튼 활성 + 취소 버튼 활성 |
| 변경기한 경과 | 일정변경 버튼 비활성 (이유 문구 표시) + 취소 버튼 활성 |
| 모든 기한 경과 | 두 버튼 모두 비활성 (이유 문구 표시) |
| 강제배정 코스 | 관리자 정책 우선 (버튼 표시 여부는 관리자 설정) |

**비활성 이유 문구**:
- 일정변경 불가: "일정 변경 기한이 지났어요"
- 취소 불가: "취소 기한이 지났어요"

### 5.2 취소 완료 화면 (app-cancelled)

- 취소 사유 텍스트 입력 → 확인 모달 → 취소 완료 화면
- 취소 완료 화면 구성: 체크 아이콘 + "수강 취소가 완료됐어요" + [코스 다시 보기] CTA

## 6. MY 오프라인 탭 — 히어로 카드 5가지 뷰 상태

| 뷰 상태 | 조건 | 히어로 노출 | 설명 |
|---------|------|------------|------|
| `D-7이상` | 다음 수업 D-7 초과 | 없음 (일반 목록만) | 히어로 불필요 시점 |
| `D-3일반` | 다음 수업 D-3~D-7 | 파란 그라데이션 히어로 | D-Day 카운트다운 |
| `오늘교육` | 수업 당일 (라이브 미시작) | 파란 히어로 + QR CTA | QR 체크인 유도 |
| `라이브모드` | 관리자가 라이브 시작 후 | 빨간 그라데이션 + 펄스 애니메이션 | 실시간 상태 |
| `수강코스없음` | 신청 코스 없음 | 없음 (빈 상태) | "코스 탐색하기" CTA |

### 6.1 바쁜 주 뷰 (이번 주 5건 이상)

- 메인 히어로 1개 + 컴팩트 업커밍 리스트 (D-Day 칩 + 코스명 + 시간)
- 컴팩트 카드 최대 4개 노출, 초과 시 "+N개 더보기"

## 7. QR 출석 체크인 — 결과 5종 명세

| 결과 타입 | 코드 | 아이콘 | 배경색 | 제목 | 부제 |
|----------|------|--------|--------|------|------|
| 출석 완료 | `qr-success` | ✓ (초록) | `#22C55E` | "출석 완료!" | "오늘도 화이팅이에요" |
| 지각 | `qr-late` | ⏰ (주황) | `#F97316` | "지각 처리됐어요" | "다음엔 일찍 와요!" |
| 체크인 시간 종료 | `qr-expired` | ✗ (회색) | `#6B7280` | "체크인 시간이 지났어요" | "강사에게 문의해주세요" |
| 수강 대상 아님 | `qr-notarget` | ⚠ (회색) | `#6B7280` | "수강 대상이 아니에요" | "현재 회차 수강생이 아닙니다" |
| 수업 시작 전 | `qr-notyet` | 🕐 (파랑) | `#5569FF` | "아직 시작 전이에요" | "수업 시작 후 다시 스캔해주세요" |

> 모달 닫기 딜레이: success/late는 3초 자동 닫힘, 나머지는 수동 닫기

## 8. 라이브 모드 — 관리자↔학습자 연동 명세

### 8.1 BroadcastChannel 이벤트

| 이벤트 | 발신 | 수신 | 처리 |
|--------|------|------|------|
| `LIVE_MODE_START` | `offline-attendance.html` | `my.html` | 히어로 카드 → 빨간 라이브 모드로 전환 |
| `LIVE_MODE_END` | `offline-attendance.html` | `my.html` | 히어로 카드 → 일반 D-Day 모드로 복귀 |

### 8.2 라이브 모드 히어로 UI

- 빨간 그라데이션 배경 (`#FF5555` → `#FF8800`)
- 우측 상단: 빨간 펄스 점 + "LIVE" 텍스트
- QR 체크인 버튼 (Primary CTA)
- 지도 버튼 (Secondary) — Phase2에서 실제 지도 연동, Phase1은 버튼 노출만
