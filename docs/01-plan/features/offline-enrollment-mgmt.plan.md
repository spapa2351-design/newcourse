## Executive Summary

| 관점 | 내용 |
|------|------|
| **Problem** | 오프라인 교육 현장에서 출석 확인이 수기 명단 체크로 이뤄져 집계 오류·누락 발생. 회차 마감 후 수료 처리가 HRD 담당자의 수동 작업에 의존. 수강 탭에서 차수 전환 UI 없어 학습자별 출석 확인이 불편 |

| **Core Value** | "현장에서 QR로 출석하고, 수강 탭에서 차수별 학습자를 확인하고, 회차 마감 후 수료까지 화면에서 완결" |

## 1. 사용자 인텐트 발견

### 핵심 문제
오프라인 교육의 출석 확인·집계·수료 처리가 수기 방식으로 운영되어 오류·누락이 발생하고 HRD 담당자의 수작업 부담이 큼. admin1-1.html 원본 버전은 회차 마감·수료 처리·수강 탭 차수 전환이 미구현 상태.

### 주요 사용자

| 사용자 | 역할 | 핵심 니즈 |
|--------|------|-----------|
| **HRD 담당자** | 출석 수정, 회차 마감, 수료 처리, 통계 확인 | 수동 개입 최소화, 한눈에 집계, 차수 전환 용이 |
| **강사** | 현장 QR 화면 운영, 실시간 출석 확인 | 빠른 QR 진입, 실시간 현황 |
| **상위 관리자** | 차수별 출석률·수료율 리포트 | 집계 자동화 |

### 성공 기준
- 강사가 라이브 모드로 QR을 띄우면 학습자 스캔 즉시 출석 반영
- HRD 담당자가 회차 마감 → 미확인 일괄 결석 전환 → 수료 처리까지 화면에서 완료
- 수강 탭에서 차수별 버튼 클릭으로 학습자 목록 전환
- 차수별 출석률·수료율이 자동 집계되어 별도 엑셀 불필요

## 2. Phase별 범위 확정

### ✅ Phase 1 — 릴리즈 범위

| 기능 | 설명 |
|------|------|
| 요약 탭 | 코스 개요, 오늘/다음 회차, 전체 출석률, 알림 패널 |
| 수강 탭 | 차수 탭 전환 버튼, 수강생 목록, 출석률 표시 |
| 출석 탭 | 회차별 출석 명단, 5종 상태 수동 변경, 회차 마감 버튼 |
| 라이브 QR 모드 | 전체화면 QR + 30초 자동갱신 + 실시간 사이드바 |
| 회차 마감 | 미확인 → 결석 일괄 전환 (handleCloseSession) |
| 수료 처리 | 수료 조건 충족 학습자 일괄 수료 확정 (handleCompletion, 75% 기준) |
| 통계 대시보드 | 차수별 출석률 바차트, 전체 수료율 |
| 캘린더 뷰 | 월별/주별 회차 일정 시각화 |
| 퍼블리싱 가드 | 미퍼블리싱 코스 접근 시 안내 화면 |
