# 온라인 코스 등록 빌더 — 공식 기획 Plan

---

## Executive Summary

| 관점 | 내용 |
|------|------|
| **Problem** | HRD 담당자가 온라인 코스 개설 시 기본정보·커리큘럼·수료조건·수강기간을 별도 시스템에 수동 입력. 콘텐츠 이수조건 연결이 분산되어 설정 오류 발생 |
| **Solution** | 단일 화면 4섹션 폼(기본정보·신청설정·수강기간·수료조건) + 우측 커리큘럼 사이드바로 챕터→항목 구성. 라이브러리에서 콘텐츠 검색 후 이수조건 즉시 설정 |
| **기능/UX 효과** | 퍼블리싱 전 6개 체크리스트 자동 검증. 임시저장 토스트 피드백. 드래그앤드롭으로 커리큘럼 순서 재배치 |
| **Core Value** | "코스 기획에서 퍼블리싱까지 한 화면에서" — offline-builder와 함께 LMS 코스 등록의 온라인 축을 완결 |

|------|------|
| **WHY** | 온라인 코스 개설 프로세스 디지털 통합 — 분산된 입력 화면을 단일 빌더로 대체 |

## 1. 사용자 인텐트 발견

### 핵심 문제
온라인 코스 개설 시 기본정보 입력, 커리큘럼 구성, 이수조건 설정, 수강기간 설정이 분산된 화면에서 이루어져 누락·오류 발생.

### 주요 사용자

| 사용자 | 역할 | 핵심 니즈 |
|--------|------|-----------|
| **HRD 담당자** | 코스 기획, 커리큘럼 구성, 퍼블리싱 | 한 화면에서 빠른 코스 개설 |
| **콘텐츠 관리자** | 라이브러리 영상/시험/설문/아티클 관리 | 등록된 콘텐츠를 코스에 쉽게 연결 |
| **수강생** | 코스 수강 | 명확한 수료 조건 안내 |

### 성공 기준
- HRD 담당자가 신규 온라인 코스를 기본정보 → 커리큘럼 → 수료조건 → 퍼블리싱 순으로 단일 화면에서 완료
- 라이브러리에서 콘텐츠 검색 후 즉시 커리큘럼에 추가, 이수조건 설정
- 퍼블리싱 전 6개 체크리스트 자동 검증으로 불완전 코스 공개 방지

## 2. Phase별 범위 확정

### ✅ Phase 1 — 릴리즈 범위

| 기능 | 설명 |
|------|------|
| 코스 기본 정보 | 코스명, 카테고리, 난이도(입문/중급/고급), 강사, 예상학습시간, 강사소개, 코스소개, 수료증발급 토글 |
| 수강 신청 설정 | 신청기간(datetime-local), 등록방식(자율수강/강제할당), 취소정책(자율수강 시) |
| 수강 기간 설정 | 고정기간 / 신청 후 N일 / 무제한 3가지 모드 |
| 수료 조건 설정 | 필수 콘텐츠 전체 이수(고정) + 선택 콘텐츠 N개 이수 옵션 |
| 커리큘럼 관리 | 챕터 → 항목 2계층 구조, 드래그앤드롭, 라이브러리 가져오기, 직접 업로드 |
| 항목 이수조건 설정 | 영상(시청률%), 시험(합격점수/문항수), 설문/아티클(고정) |
| 퍼블리싱 검증 | 6개 체크리스트 실시간 검증, 퍼블리싱 확인 모달 |
| 임시저장 | 상태 관리(draft/saved/published), 토스트 피드백 |
| 퀵 스탯 바 | 코스유형/전체항목수/필수콘텐츠수/완성도% |

## 3. 아키텍처 결정사항

### 레이아웃

```
[헤더] ← 뒤로가기 | 코스등록 > 🎬온라인 | 상태뱃지 | [임시저장] [미리보기] [퍼블리싱]

[좌측 메인 flex-1]              [우측 사이드바 500px]
 퀵 스탯 바 (4개 카드)          커리큘럼 관리 헤더 (dark)
 섹션 1: 코스 기본 정보         챕터 목록 (DnD)
 섹션 2: 수강 신청 설정           └── 챕터 헤더 + 항목 리스트
 섹션 3: 수강 기간 설정               └── 항목 카드 (DnD)
 섹션 4: 수료 조건 설정         하단: 체크리스트 + 퍼블리싱 버튼
```

### 슬롯(항목) 유형 및 이수 조건

| 유형 | 기본 이수조건 | 설정 가능 |
|------|-------------|---------|
| 영상(video) | 시청률 90% 이상 | 시청률 슬라이더 50~100% |
| 시험(exam) | 60점 이상 / 20문항 | 합격점수, 총 문항수 |
| 설문(survey) | 전 문항 응답 후 제출 | 고정 (변경 불가) |
| 아티클(article) | 전체 열람 완료 | 고정 (변경 불가) |

### 기술 스택
- React 18 (Babel standalone, ESM CDN)
- Tailwind CSS (CDN)
- Lucide React 0.292.0
- 단일 HTML 파일 (백엔드 없음)

## 4. 퍼블리싱 검증 체크리스트

| 항목 | 조건 |
|------|------|
| 코스명 입력 | `title.trim()` 존재 |
| 담당 강사 지정 | noInstructor 또는 instructor 입력 |
| 신청 기간 설정 | start + end 모두 입력 |

| 필수 콘텐츠 최소 1개 | `reqSlots.length > 0` |

## 5. 수강 신청 방식 타입 분석

> 출처: `docs/guide-offline-lms.html` — 학습자 가이드 "수강 신청" 섹션 기준

### 4가지 수강 신청 방식 개요

| 타입 | 키 | 배지 | 개요 |
|------|-----|------|------|
| 자율 단일 선택 | `single` | 🔵 blue | 학습자가 커리큘럼 1개를 직접 선택 |
| 자율 다중 조합 | `multi` | 🟣 violet | 운영자 설정 개수(N개)만큼 자유 조합 선택 |
| 관리자 강제배정 | `assigned` | 🟡 amber | 운영자가 지정, 학습자는 확인만 가능 |
| 날짜선택형 회차 | `sessionSelect` | 🟢 green | 커리큘럼 선택 후 캘린더에서 회차 날짜 선택 |

---

### ① 자율 단일 선택 (`single`)

**플로우**
```
코스 상세 열기 → "수강 신청하기" 터치 → 일정 1개 선택 → 신청 확인 → 완료 모달
```

| 화면 단계 | 설명 |
|----------|------|
| 일정 선택 모달 | 커리큘럼 목록 표시. 선택 전엔 버튼 비활성. 잔여석 배지(여유/마감임박/마감) 표시 |
| 일정 선택 완료 | 선택 카드 하이라이트. "신청 확인" 버튼 활성화 |
| 신청 확인 화면 | 선택 일정 요약·장소 표시. 최종 확인 후 신청 |

---

### ② 자율 다중 조합 (`multi`)

**플로우**
```
코스 상세 열기 → "수강 신청하기" 터치 → N개 복수 선택 → 신청 확인 → 완료
```

| 화면 단계 | 설명 |
|----------|------|
| 다중 선택 모달 | 체크박스 방식. 상단에 "0 / N개 선택" 카운터 표시 |
| N개 선택 완료 | 설정 개수 충족 시 버튼 활성. 선택한 카드에 체크 표시 |
| 신청 확인 | 각 커리큘럼 날짜·시간·장소를 개별 카드로 나열 |

> **조건**: 설정 개수를 모두 채워야 "신청 확인" 버튼 활성화

---

### ③ 관리자 강제배정 (`assigned`)

**플로우**
```
코스 상세 열기 → 배정 안내 배너 확인 → 배정 일정 확인
```

| 화면 | 설명 |
|------|------|
| 코스 상세 (배정됨) | 🛡️ "관리자 배정 교육" 배너. 버튼: "배정 일정 확인 완료" (비활성) |
| 배정 일정 확인 | 모달 없이 바로 배정 내역 확인. 그룹명·날짜·장소 표시 |

> **조건**: 학습자가 일정을 선택할 수 없음. "수강 신청하기" 버튼 자체가 비활성 상태

---

### ④ 날짜선택형 회차 (`sessionSelect`)

**플로우**
```
코스 상세 열기 → 커리큘럼 선택 → 캘린더 날짜 선택 → 회차 선택 → 신청 확인
```

| 화면 단계 | 설명 |
|----------|------|
| 커리큘럼 선택 → 캘린더 | 날짜선택형 카드 선택 시 캘린더 UI가 하단에 펼쳐짐 |
| 캘린더 날짜 선택 | 파란 점(●) 있는 날짜에 수업 있음. 날짜 선택 시 해당 날의 회차 목록 표시 |
| 회차 선택 완료 | 시간·장소 확인 후 선택. "신청 확인" 버튼 활성 |
| 신청 확인 | 선택 회차 날짜·시간·장소가 확인 화면에 표시 |

> **주요 사용 케이스**: 법정의무교육 등 날짜 선택형 운영

---

### 신청 기간별 상태 (미신청자 기준)

| 상태 | 커버 배지 | 하단 버튼 |
|------|----------|----------|
| 신청 오픈 전 | 🟡 오픈 예정 | 오픈 일시 안내 (비활성) |
| 신청 기간 중 | 🔵 신청 가능 | "수강 신청하기" (활성) |
| 신청 마감 | ⬜ 신청 마감 | "신청이 마감되었습니다" (비활성) |
| 수강중 | 🟢 수강중 | "내 수강 상세 보기 →" (활성) |
| 수강 완료 | ⬜ 수강 완료 | 수강 완료 · 출석률% (비활성) |
