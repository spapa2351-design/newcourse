# 강의 코스 목록 — 기획 Plan

> **UI 라벨**: "강의 코스" (코드 상수: `CONTENT`). 파일명: `admin/online-list.html`.
> **상위 차터**: 메모리 `project-newcourse-charter` + `project-newcourse-course-system-architecture`.

## Executive Summary

| 관점 | 내용 |
|---|---|
| **Problem** | 온라인 코스가 늘면 HRD 담당자가 한 곳에서 목록·상태·검색·신규 등록을 처리할 화면 필요. 현재는 빌더만 있고 목록 페이지 부재. |
| **Solution** | legacy 코스 목록 셸(admin/offline-list.html) 그대로 차용 + 온라인 특성 반영(차수 제거, "콘텐츠"→"커리큘럼"으로 의미 정합, 필수 칩을 코스명 prefix로 흡수) |
| **기능/UX 효과** | 운영팀 학습 곡선 0 — 익숙한 컬럼·필터·분류 트리 그대로. 1,000+ 코스 페이지네이션 처리. |
| **Core Value** | "온라인 코스 운영 한 곳에서 — legacy 익숙함 유지하며 newcourse 모델로 정합" |

## Context Anchor

| 항목 | 내용 |
|---|---|
| **WHY** | 빌더만 있고 목록 부재 → 운영자가 코스 관리·신규 등록 진입점 없음. legacy 사용자 마이그레이션을 위해 친숙한 셸 유지 필요. |
| **WHO** | HRD 담당자, 콘텐츠 관리자, 코스 운영자 |
| **RISK** | legacy 사용자가 컬럼 변경(콘텐츠→커리큘럼)에 혼란 / 1000+ 코스 렌더링 성능 / 필터·검색 인덱스 부재 |
| **SUCCESS** | 운영팀이 legacy처럼 자연스럽게 사용. 신규 등록 → 빌더 진입 흐름 단일 클릭. 1000+ 코스에서도 페이지네이션으로 부드럽게 동작. |
| **SCOPE** | 프론트엔드 단일 HTML 프로토타이핑. 백엔드 미연결, mock 데이터 |

## 한 줄 룰

> **"legacy 코스 목록의 익숙함을 유지하면서 newcourse 모델(코스→커리큘럼→에셋)로 의미를 정합한다."**

### 5가지 이유

1. **학습 곡선 0** — legacy 컬럼·분류 트리·CTA 위치 그대로 차용. 운영팀 재학습 X.
2. **의미 정합** — "콘텐츠" 컬럼이 newcourse 위계와 어긋남 → "커리큘럼 N · 에셋 M"으로 통일.
3. **공간 효율** — "필수 여부" 별 컬럼은 80% 빈 셀 → 코스명 prefix 칩으로 흡수.
4. **성능** — 1000+ 코스도 20개/페이지로 페이지네이션. 무한 스크롤 X (메모리 부담·페이지 번호 해석 어려움).
5. **단일 진입** — [등록] → online-builder.html 직행. 코스 유형 모달 생략 (온라인 목록에서 등록은 항상 온라인).

## 1. Phase 범위

### Phase v1 — 현 작업 범위

| 영역 | 기능 |
|---|---|
| **레이아웃** | admin/offline-list.html 셸 그대로 (좌측 사이드바 + 탑네비 + 분류 트리 + 메인 카드) |
| **분류 트리** | legacy 차용 (업무 역량·WORK·직무 역량·상품 가이드·게이미피케이션·IT 등 mock) |
| **상단 툴바** | 상태 탭 + 필터 칩 + [+ 필터 추가] + 초기화 |
| **카운트·검색·뷰** | 총 N개 + 검색(코스명) + 일괄선택 카운트 + [다운로드 mock] + [그리드/리스트 토글] |
| **테이블** | 10컬럼 디폴트 (아래 참조) |
| **상태 5종** | 초안 / 운영중 / 수강중 / 종료 / 관리자 종료 (학습자 측은 "신청 마감"으로 표시) |
| **페이지네이션** | 20개/페이지 + 이전/다음 |
| **정렬 디폴트** | 등록일 최신순 (legacy "코스 순서 설정순"은 v2) |
| **CTA** | [등록] → `online-builder.html` 직행 / [수강생 등록] mock / [코스 순서 관리] v2 |
| **일괄 액션** | 일괄 삭제만 (1차) |
| **신규 코스 등록 시점** | 1차는 mock 데이터, 백엔드 미연결 |

### Phase 후속 (별도 작업)

- 코스 순서 관리 (드래그·드롭 정렬, "코스 순서 설정순" 정렬)
- 컬럼 토글 (생성자·신청 기간 등 디폴트 숨김 컬럼 표시)
- 일괄 종료 / 일괄 공개 / 일괄 분류 변경
- 그리드 뷰 토글 (현재는 리스트만)
- 다국어 코스 분리 표시
- 수강생 등록 모달 실 동작 (현재 [수강생 등록] 버튼 placeholder)

## 2. 컬럼 설계 (10개 디폴트)

| # | 컬럼 | 폭 | 내용 |
|---|---|---|---|
| 1 | ☐ | 40px | 체크박스 (전체선택 헤더 포함) |
| 2 | 상태 | 80px | 5종 칩 (초안·운영중·수강중·종료·관리자 종료) |
| 3 | 표지 | 52px | 작은 썸네일 (40×60 portrait) |
| 4 | 코스명 | flex | bold + prefix `[필수]` 칩(분류=필수일 때만) + 하단 등록 채널 미세 라벨(관리자/자율/자동) |
| 5 | 커리큘럼 | 120px | "커리큘럼 N · 에셋 M" 요약 칩 (legacy "콘텐츠" 컬럼 대체) |
| 6 | 수강관리 | 90px | [수강관리] 버튼 (자체 컬럼, 항상 노출) |
| 7 | 분류 | 90px | 분류명 (단일 칩) |
| 8 | 수강 기간 | 140px | "제한 없음" 또는 날짜 범위 |
| 9 | 수강 인원 | 70px | N명 |
| 10 | 등록일 | 130px | 날짜 + 시간 |
| 11 | 관리 | 80px | 링크·편집·삭제 아이콘 3종 |

> 11번째 "관리"는 디폴트 노출이지만 폭 좁게. 생성자 컬럼은 1차 제외 (v2 토글).

## 3. 상태 5종 정의

| 상태 | 정의 | 칩 색상 |
|---|---|---|
| **초안** | draft. 퍼블리시 안 됨. 운영자에게만 보임. | slate |
| **운영중** | published + 신청 기간 진행 중. 신규 등록 가능. | emerald |
| **수강중** | 신청 기간 끝남. 수강만 진행. 신규 등록 불가. (학습자 측 라벨: "신청 마감") | amber |
| **종료** | 수강 기간까지 끝남. 학습 종료. | slate (dim) |
| **관리자 종료** | 관리자가 강제 종료. 진행 중이었어도 즉시 차단. | rose |

## 4. 데이터 모델

```
Course {
  id, title, coverImage,
  status: 'draft' | 'running' | 'closed_apply' | 'ended' | 'admin_ended',
  classification: 'normal' | 'mandatory',  // 일반 / 필수
  category: string,                         // 분류
  curriculumCount, assetCount,              // 커리큘럼·에셋 요약
  enrollMode: 'admin' | 'self' | 'auto',    // 등록 채널
  studyPeriod: { type: 'fixed'|'days'|'unlimited', start?, end?, days? },
  applyPeriod: { start?, end? },
  enrolledCount: number,
  createdAt, createdBy,
}
```

### 1차 schema에 박을 v2 확장 필드 여지

| 필드 | 1차 | v2 |
|---|---|---|
| `order` | null | 코스 순서 설정 정수 |
| `tenant_id` | null | 멀티테넌트 시 출처 |
| `archived` | false | 아카이브 처리 시 true |

```

   ↓

```

## 6. 검색·필터

| 영역 | 1차 |
|---|---|
| 검색 | 코스명 (text) |
| 상태 탭 | 5종 (초안·운영중·수강중·종료·관리자 종료) + 전체 |

## 11. 후속 작업 후보

| 작업 | 우선순위 | 비고 |
|---|---|---|
| 코스 순서 관리 (드래그) | High | legacy 디폴트 정렬을 newcourse에 흡수하려면 필요 |
| 컬럼 토글 (생성자 등) | Medium | 데이터 누적 후 |
| 일괄 종료·공개·분류 변경 | Medium | 운영 케이스 더 쌓이면 |
| 그리드 뷰 토글 | Low | 표지가 중요한 코스에 한정 |
| 수강생 등록 모달 실 동작 | Medium | 별도 기능 연동 |
| 다국어 코스 분리 표시 | v2+ | 다국어 도입 시 |
| 빈 목록 첫 사용자 가이드 | Low | 신규 tenant 온보딩 |
