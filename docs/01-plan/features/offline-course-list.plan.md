## Executive Summary

| 관점 | 내용 |
|------|------|
| **Problem** | 오프라인 교육 코스가 늘어남에 따라 관리자가 코스별 차수·장소·수강 현황을 한눈에 파악하기 어렵고, 상태별/분류별 필터링 없이 전체 목록을 스크롤해야 하는 비효율이 존재한다. |
| **Solution** | 3패널(글로벌 사이드바 + 분류 사이드바 + 테이블 메인)로 구성된 관리 페이지를 제공하여 상태 탭 필터·검색·카테고리 트리를 조합해 원하는 코스를 즉시 찾고, 행 확장으로 차수 상세를 인라인 확인할 수 있게 한다. |
| **기능 UX 효과** | 코스 탐색 시간 단축, 차수별 잔여 인원 즉시 확인, 수강생 등록/코스 편집 워크플로를 1클릭으로 진입 가능하여 담당자 운영 효율이 향상된다. |
| **Core Value** | "찾기 → 확인 → 진입"을 단일 화면에서 해결하는 오프라인 코스 운영 허브. |

## Context Anchor

### WHY (배경)
오프라인 코스는 온라인 코스와 달리 패키지(반) 단위 일정·장소·정원이 복잡하게 얽혀 있다. 기존 단순 목록 UI로는 이 구조를 표현하기 어려워 별도 특화 목록 뷰가 필요하다.

### WHO (대상 사용자)
- **Primary**: LMS 관리자(HR 담당자) — 코스 등록·수정·수강생 관리 전담
- **Secondary**: 운영 담당자 — 수강 현황·잔여석 모니터링

### RISK (위험 요소)
| 위험 | 완화 방안 |
|------|----------|
| 더미 데이터 의존 (백엔드 없음) | localStorage 연동으로 커버 이미지만 영속화, 나머지는 COURSES 상수로 관리 |
| CDN 의존성 (Babel Standalone 포함) | 프로덕션 전환 시 번들러 도입 필요 — Phase 2 이전 항목으로 분류 |
| Grid 뷰·다운로드 미구현 버튼 UX 혼란 | Phase 2 이월 명시, 현재 버튼은 시각적 플레이스홀더로만 존재 |

### SUCCESS (성공 기준)
- [ ] 상태 탭 필터(전체/운영중/임시저장/종료) 정확하게 동작
- [ ] 코스명/강사명 검색 실시간 반영
- [ ] 분류 사이드바 카테고리 클릭 시 테이블 필터링
- [ ] 행 확장 클릭 시 차수 상세(패키지별 기간/장소/회차/정원/신청/잔여) 표시
- [ ] selectedIds > 0 시 선택 삭제 버튼 노출 및 일괄 삭제 동작
- [ ] 커버 이미지 localStorage 연동 (admin1.html에서 등록한 이미지 표시)

### SCOPE (범위)
- **In-Scope (Phase 1)**: 목록 조회, 상태 탭 필터, 검색, 카테고리 필터, 차수 확장 행, 코스등록 링크, 수강관리 링크, 커버이미지 표시, 단건 삭제, 선택 삭제(일괄)
- **Out-of-Scope (Phase 2)**: Grid 뷰 토글, 다운로드 기능, 코스 순서 관리, 추가 필터 조건

## Phase 1 범위 (MVP)

| # | 기능 | 설명 |
|---|------|------|
| F-01 | 목록 조회 | COURSES 더미 데이터 기반 테이블 렌더링 |
| F-02 | 상태 탭 필터 | 전체/운영중/임시저장/종료 + 카운트 뱃지 |
| F-03 | 검색 | 코스명·강사 실시간 텍스트 필터 |
| F-04 | 카테고리 필터 | 분류 사이드바 트리 클릭 필터링 |
| F-05 | 차수 확장 행 | 패키지별 기간/장소/회차/정원/신청/잔여 인라인 표시 |
| F-06 | 코스등록 링크 | 헤더 "코스 등록" → admin1.html |
| F-07 | 수강관리 링크 | 운영중/종료 코스 행 "수강관리" → admin1-1.html |
| F-08 | 커버이미지 | localStorage에서 lms_cover_{id} 로드 후 표지 셀 표시 |
| F-09 | 단건 삭제 | Trash2 아이콘 클릭 → confirm → setCourses |
| F-10 | 선택 삭제 | selectedIds > 0 조건부 버튼 → confirm → 일괄 setCourses |
