> **분석 범위**: 고객사 전달 스펙 이미지 기준 Phase 1·2 기능만 대상. Phase 3(학습이력 통합·수료기준), Phase 4(알림·통계 고도화)는 제외.

## 1. 사용자 환경 (User)

### ① 교육 일자 선택 및 신청 기능 — `△ 부분 충족`

| 항목 | 고객 요청 | 현재 기획 | 판정 |
|------|-----------|-----------|------|
| 차수별 선택 | 회차별·일자별 선택 UI | FLOW-02: 차수 목록(날짜/시간/잔여석) 확인 후 신청 | ✅ |
| 날짜 조합 선택 | 필요 교육 클릭 → 내부에서 회차·일자를 직접 선택 | **명세 없음** — 현재는 "차수 1개 신청" 단위. 한 코스 내 여러 날짜 조합 선택 플로우 미정 | ❌ |

**Gap 설명**: "이번 달 내 원하는 날짜 회차를 직접 골라서 신청"하는 날짜-피커형 신청 UX가 기획서에 없음. 현재는 차수 단위 신청 구조.

|------|-----------|-----------|------|
| 신청 내역 확인 | 마이페이지에서 코스·일정 확인 | FLOW-02 자율 수강 신청 플로우 정의 | ✅ |
| 일정 변경 | 같은 코스에서 다른 차수로 변경 | 섹션 2: 일정 변경 제한 토글 설정 | ✅ |
| 수강 취소 | 취소 후 "취소완료" 문구 노출 | 취소 정책 3단계(언제든/기한부/불가) | ✅ |
| UX 카피 | "취소완료라고 뜨도록" | 현재 명세에 정확한 카피 미정 | △ |

## 2. 관리자 환경 (Admin)

### ① 교육일 D-1·D-day 알림 자동화 — `△ 부분 충족`

| 항목 | 고객 요청 | 현재 기획 | 판정 |
|------|-----------|-----------|------|
| 알림 기능 포함 여부 | Phase 2 포함 | Phase 2 항목에 "알림 채널 확정 + 리마인더 자동" 포함 | ✅ |
| 알림 채널 | 앱 푸시 명시 | **GAP-03**: 앱 푸시/이메일/SMS 결정 안 됨 | ❌ |
| 미출석자 자동 알림 | 시작 5분 전, 미체크 인원만 자동 발송 | 세부 로직 미정 | ❌ |
| 관리자 커스텀 멘트 | 관리자가 설정한 멘트로 발송 | 명세 없음 | ❌ |

**Gap 설명**: Phase 2에 넣기로 되어 있으나 알림 채널, 발송 타이밍 로직, 커스텀 멘트 요구사항이 기획서에 미반영. GAP-03 해결 필요.

### ② 수강 신청 리스트·일자별 최종 참석 리스트 확인 — `△ 부분 충족`

| 항목 | 고객 요청 | 현재 기획 | 판정 |
|------|-----------|-----------|------|
| 수강 신청 리스트 확인 | 관리자 페이지에서 확인 | course-learners.html 수강생 관리 탭 | ✅ |
| 일자별 참석 리스트 | 코스 내 일자별 출석체크 리스트 | Phase 1은 코스 당 QR 1개 — 일자(회차)별 개별 리스트 명세 부족 | ❌ |
| 미신청자 QR 스캔 처리 | 신청 안 한 인원이 QR 스캔 시 관리자 확인 가능 | 현재는 신청 완료자 기준 출석 처리만 명세 | ❌ |

|------|-----------|-----------|------|

| 임시 코드 생성 | ID 없는 인원 → 임시 코드 생성 후 추후 ID 변경 | 명세 없음 (수기 체크로 대체 가능 수준) | △ |

### ④ 일자별·그룹별 정원 설정 — `❌ 미정`

| 항목 | 고객 요청 | 현재 기획 | 판정 |
|------|-----------|-----------|------|
| 일자별 개별 정원 | 날짜마다 다른 장소·정원 설정 | 차수 단위 정원 설정만 존재. 회차별 별도 정원 없음 | ❌ |
| 그룹별 인원 제한 | 2/6분단, 목요그룹 각 N명 제한 | **그룹 개념 자체가 기획서에 없음** | ❌ |
| 그룹 초과 신청 차단 | 해당 그룹 정원 도달 시 추가 신청 불가 | 미정 | ❌ |

**Gap 설명**: 이번 분석에서 가장 큰 Gap. "그룹별 인원 제한 신청" 기능은 현재 기획 구조(코스→차수→회차)에 **그룹(Group) 레이어가 없기 때문에** 데이터 모델부터 설계 필요. Phase 2 착수 전 아키텍처 결정 요구.

| 관리자 수동 마감 | 특정 상황에서 수동 마감 처리 | 4.5: 취소 정책 + 마감 개념 존재 | ✅ |
| 마감 후 대기자 플로우 | 마감 후 대기 신청·관리 | **GAP-01 미결**: 대기 신청 받을지 여부 결정 안 됨 | ❌ |

## 3. Phase 2 추가 기획 필요 항목

Phase 1·2 범위에서 고객 요청을 충족하려면 아래 항목을 Phase 2 기획에 추가해야 합니다.

| 우선순위 | 추가 기획 항목 | 관련 기존 Gap | 비고 |
|---------|--------------|:------------:|------|
| 🔴 Critical | **그룹별 정원 설정** — 데이터 모델에 Group 레이어 추가 | 신규 | 아키텍처 결정 필요 |
| 🔴 Critical | **알림 채널·커스텀 멘트 설정** — 앱 푸시/이메일 채널 결정 | GAP-03 | 채널 미결 |
| 🟠 Important | **날짜 피커형 신청 UI** — 회차 단위가 아닌 일자별 직접 선택 플로우 | 신규 | UX 설계 필요 |
| 🟠 Important | **미신청자 QR 스캔 처리 플로우** — 현장 당일 신청 케이스 | 신규 | 정책 결정 필요 |
| 🟡 Minor | **취소 완료 UX 카피** — "취소완료" 문구 및 상태 표시 명세 | 기존 | 명세 추가만 |

| 항목 | 고객 스펙 Phase | 우리 기획 Phase |
|------|:--------------:|:--------------:|
| 온·오프라인 학습 이력 통합 | Phase 3 | Phase 3 (일치) |

| 교육 통계·분석 대시보드 | Phase 4 | Phase 3 장기 |
| 알림 고도화 (자동화 완성) | Phase 4 | Phase 2 일부 + Phase 3 |

## 5. 결론

- **Phase 1 완료분**은 고객 요청 기능의 약 **70% 커버** (출석·정원·QR·수료 핵심 플로우)
- **Phase 2**는 고객 요청 기준으로 현재 계획보다 **그룹 관리·알림 상세·날짜 피커 UX** 3개 항목 추가 필요
- 가장 큰 리스크: **그룹별 정원 설정**은 현재 데이터 모델에 없는 새 레이어 — Phase 2 착수 전 아키텍처 결정 필수
- GAP-01(대기자 플로우), GAP-03(알림 채널)은 고객 요청에서도 명시된 기능이므로 **Phase 2 Critical 결정 사항**으로 격상
