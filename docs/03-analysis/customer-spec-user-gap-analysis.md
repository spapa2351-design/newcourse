## 1. 완전 구현된 기능 ✅

### ① 교육 일자 선택 및 수강 신청 UI — `course-detail.html`

| 고객 요청 | 구현 상태 |
|-----------|-----------|
| 회차별·일자별 선택 UI | ✅ 단일선택·다중조합·관리자배정 3가지 모드 구현 |
| 차수 목록 (날짜·시간·잔여석) | ✅ 목록형 + 캘린더형 선택 UI 모두 구현 |
| 신청 → 확인 → 완료 플로우 | ✅ 모달 기반 3단계 완전 구현 |
| 정원 마감 표시 | ✅ "마감" 배지 + 비활성 처리 구현 |

### ② 마이페이지 기반 신청 내역 확인 — `my.html`

| 고객 요청 | 구현 상태 |
|-----------|-----------|
| 수강중 코스 목록 | ✅ 출석률 바 + D-Day 다음회차 정보 |
| 수강예정 코스 표시 | ✅ 예정 코스 별도 구분 표시 |
| 완료된 코스 목록 | ✅ 수료/미수료 배지 구분 |
| 필터 (전체/수강중/수강예정/종료) | ✅ 4단계 필터 칩 구현 |

### ③ 정원 및 잔여석 실시간 표시 — `course-list.html`, `course-detail.html`

| 고객 요청 | 구현 상태 |
|-----------|-----------|
| 목록에서 상태 배지 (신청가능/수강중/종료) | ✅ 6가지 상태 배지 구현 |
| 코스 상세에서 잔여석 표시 | ✅ 신청 가능 좌석 UI 구현 |
| 마감 시 신청 버튼 비활성 | ✅ 구현됨 |

### ④ 일정 변경 기능 — `course-enrolled.html`

| 고객 요청 | 구현 상태 |
|-----------|-----------|
| 일정 변경 버튼 | ✅ 바텀시트로 구현 |
| 같은 코스 다른 차수 선택 | ✅ 패키지(차수) 선택 UI |
| 변경 완료 모달 | ✅ 변경 확인 → 완료 모달 |

### ⑤ 수강 취소 기능 — `course-enrolled.html`

| 고객 요청 | 구현 상태 |
|-----------|-----------|
| 취소 버튼 | ✅ 구현 |
| 취소 확인 팝업 | ✅ 모달 기반 확인 구현 |
| 취소 완료 화면 | ✅ "수강이 취소되었습니다" 전용 화면 구현 |

### ⑥ QR 출석 체크 결과 화면 — `my.html`

| 고객 요청 | 구현 상태 |
|-----------|-----------|
| 출석 완료 결과 | ✅ 출석완료/지각/기간종료/대상아님/시작전 5가지 결과 모달 |
| 오늘 교육 D-Day 히어로 카드 | ✅ D-Day별 3가지 뷰 (D-7이상/D-3/오늘) |
| QR 스캔 버튼 | ✅ 히어로 카드에 QR 버튼 |

## 2. 부분 구현된 기능 △

### ① 날짜 피커형 신청 UI — `△ 부분 구현`

| 고객 요청 | 현재 상태 | Gap |
|-----------|-----------|-----|
| "필요 교육 클릭 → 내부에서 회차·일자를 직접 선택" | 차수 카드 선택 방식으로 구현됨 | 날짜 달력에서 원하는 날 클릭하는 피커형 UX는 없음 |
| 한 달 이상 일정에서 원하는 날짜 조합 선택 | 한 차수씩 선택 방식 | 여러 날짜를 체크박스로 고르는 UI 미구현 |

**스크린**: `course-detail.html`의 `screen-detail` → 캘린더는 있으나 날짜별 직접 선택(체크)이 아닌 "차수" 단위 선택.

### ② 신청 데이터 실시간 반영 — `△ 부분 구현`

| 고객 요청 | 현재 상태 | Gap |
|-----------|-----------|-----|
| 신청·취소·정원 변동 즉시 반영 | 프로토타입 UI는 즉시 반영되는 것처럼 표시 | 실제 API 연동 없음 — 정적 상태 |
| 잔여석 -1 실시간 갱신 | 신청 완료 후 "신청 완료" 표시 | 잔여석 숫자가 실시간으로 줄어드는 UI 없음 |

> 프로토타입 특성상 허용 가능한 Gap. API 연동 시 해결.

### ③ 수료 결과 확인 — `△ 부분 구현`

| 고객 요청 | 현재 상태 | Gap |
|-----------|-----------|-----|
| 수료 여부 확인 | ✅ `course-completed.html`에 수료/미수료/지각 3가지 결과 시뮬레이션 | — |
| 수료증 다운로드 | `course-completed.html`에 다운로드 버튼 존재 | 실제 PDF 생성·다운로드 기능 없음 (Phase 2 예정) |
| MY 페이지에서 수료 배지 | ✅ "수료" 배지 표시 | — |

## 3. 미구현 기능 ❌

### ① 일자 별 교육 신청하기 드롭다운 — `❌ 미구현`

고객 스펙 이미지에서 명시한 "일자 별 교육 신청하기" 드롭다운 UI:

```
일자 별 교육 신청하기  ▼
11월01일 교육 신청 [마감]
11월02일 교육 신청 [잔여:5]
11월03일 교육 신청 [잔여:11]
11월04일 교육신청 [마감]
```

| 고객 요청 | 현재 상태 |
|-----------|-----------|
| 일자별 드롭다운에서 잔여석·마감 표시 | **없음** — 현재는 차수 카드 전체 리스트 표시 방식 |
| 마감된 날 선택 불가 | 부분 — 마감 배지 있으나 드롭다운 형식 아님 |

## 4. 기존 관리자단 분석과 연계

`customer-spec-gap-analysis.md`에서 도출한 Phase 2 추가 기획 항목이 사용자단에도 연동:

| 추가 기획 항목 | 관리자단 영향 | 사용자단 영향 |
|--------------|-------------|-------------|
| 그룹별 정원 설정 | 관리자 설정 화면 | 사용자 신청 시 "그룹 초과" 안내 UI |
| 날짜 피커형 신청 UI | 관리자 일자별 정원 설정 | **사용자 신청 화면 UX 변경 필요** |
| 미신청자 QR 처리 | 관리자 확인 화면 | QR 스캔 결과 "대상 아님" → "현장 신청" 플로우 추가 |
| 취소 완료 UX 카피 | — | `course-enrolled.html` 취소 완료 화면 카피 확정 필요 |

## 5. 완성된 사용자 플로우 전체 맵

```
home.html
  ├── [오프라인 코스 카드] → course-detail.html
  ├── [수강중 카드]        → course-enrolled.html
  ├── [학습완료 카드]      → course-completed.html
  ├── [더보기 버튼]        → course-list.html
  ├── [카테고리 탭]        → category.html
  └── [MY 탭]             → my.html

category.html
  ├── [오프라인 카테고리]  → course-list.html ✅
  └── [온라인 카테고리]   → course-list.html ✅ (신규 연결)

course-list.html
  ├── [신청가능 카드]      → course-detail.html ✅
  ├── [수강중 카드]        → course-enrolled.html ✅
  ├── [수강예정 카드]      → course-detail.html ✅ (신규)
  ├── [수강종료 카드]      → course-completed.html ✅ (신규)
  └── [학습완료 카드]      → course-completed.html ✅

course-detail.html
  └── [신청완료 후]        → course-enrolled.html ✅

course-enrolled.html
  ├── [취소 완료]          → my.html ✅
  └── [일정 변경]          → (동일 페이지 내 처리)

course-completed.html
  └── [뒤로가기]           → history.back()

my.html
  ├── [수강중 카드]        → course-enrolled.html ✅
  ├── [완료 카드]          → course-completed.html ✅
  └── [코스 탐색하기]      → category.html ✅ (신규)
```

## 6. 결론

- 사용자단 프로토타입은 고객 요청 핵심 플로우의 **약 80% 완성**
- 나머지 20%는 **날짜 피커형 신청 드롭다운 UI** 1개와 **API 실시간 연동** (Phase 2)
- 네비게이션은 이번 작업으로 **전면 연결 완료** — 404 없이 전 플로우 탐색 가능
- Phase 2 착수 전 최우선 UX 결정: 날짜 피커형 vs 차수 카드형 신청 방식 선택
